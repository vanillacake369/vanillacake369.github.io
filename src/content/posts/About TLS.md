# What? 뭘 배움?

## SSL/TLS 란 ??

### SSL

SSL(Secure Sockets Layer)은 1990년대 초 넷스케이프(Netscape)사에 의해 개발되었다.
웹 브라우저와 서버 간의 데이터를 암호화하기 위한 혁신적인 시도였으나, 초기 버전들은 보안 결함이 많았다.

- **SSL 1.0:** 공개조차 되지 못한 채 내부 설계 단계에서 폐기되었다.
- **SSL 2.0:** 1995년 출시되었으나 심각한 보안 취약점이 발견되어 1년 만에 교체되었다
- **SSL 3.0:** 1996년 설계를 전면 수정해 출시되었습니다. 오랫동안 사용되었으나, 2014년 **POODLE(푸들)** 공격에 취약하다는 사실이 밝혀지며 표준으로서의 생명이 끝났다.

### TLS

이후 IETF(국제 인터넷 표준화 기구)가 SSL 3.0을 계승하여 만든 표준 프로토콜을 만들게 된다.
이것이 바로 TLS 이다.

- **TLS 1.0:** 1999년 등장했습니다. SSL 3.0과의 차이점이 미미해 'SSL 3.1'이라 불리기도 했지만, 암호화 방식이 더 정교해졌습니다.
- **TLS 1.2:** 2008년 발표되었으며, 현재 가장 널리 쓰이는 표준입니다. 강력한 해시 알고리즘(SHA-256)을 지원합니다.
- **TLS 1.3:** 2018년 확정된 최신 버전입니다. 불필요한 핸드셰이크 과정을 줄여 속도를 획기적으로 개선하고, 보안성이 낮은 암호화 알고리즘을 강제로 퇴출했습니다.

### 왜 우리는 아직도 SSL이라 부를까?

기술적으로 현재 우리가 사용하는 모든 보안 인증서는 **TLS**입니다. 하지만 'SSL'이라는 용어가 먼저 대중화되었기 때문에 관성적으로 사용되고 있습니다.
또한 과거 장비들이나 설정 메뉴에 'SSL'이라는 명칭이 남아있는 경우가 많아 혼동을 방지하기 위해 혼용합니다.

## TLS 역할

앞서 SSL/TLS 의 역사를 알아보았다. 그렇다면 TLS 란 도대체 무엇인가? 진짜 그것이 하는 일이 뭔가?
TLS 는 OSI Layer 의 6계층인 Presentation Layer 에서 작동하는 암호화 프로토콜이다.
짧고 굵게 OSI Layer 를 설명해보자면, 데이터를 송수신하는 일종의 순서에 따른 프로토콜 처리순서이며
7계층 순으로 Client 가 Server 에게 데이터를 요청하거나 받는다.
예를 들자면 다음과 같다.

1. 요청
2. ( Server 가 로직 수행 이후 반환값 준비한다. )
3. 반환

![](/images/notion/1c2554742bd8e54d.png)


이 때 6계층인 Presentation Layer 는 종단 암호화를 담당하는 계층이며, 여기서 TLS 라는 프로토콜이 작동하는 것이다.

![](/images/notion/11895a65371d8860.png)


이제 TLS 가 6계층 암호화를 담당하는 것을 이해했다면 TLS 가 어떻게 종단 간 암호화를 지원하는지 알아볼 차례이다.
TLS 암호화 처리 과정과 원리를 이해하려면 우선 암호화 과정 자체를 이해해야한다.
암호화/복호화는 세부적으로 다루면 몇 장의 블로그 포스트가 필요하다.
이를 거시적으로 알아보자.
우선 개인키와 공개키에 대해서 알아야한다.
**개인키 (Private Key) 는** 자신만 가지고 있는 **비밀키이며,** 다른 호스트와 공유하지 않는다.
생성방법은 보통 `OpenSSL` 같은 도구를 사용하며, 아래와 같은 명령어로 생성한다.
공개키를 통해 역산하는 것은 연산 성능으로 거의 불가능하다.

```bash
# 2048비트 길이의 RSA 개인키 생성
openssl genrsa -out private.key 2048
```

**공개키 (Public Key) 는** 누구나 가질 수 있는 **공개된 키이며,** 개인키와 수학적으로 한 쌍이다.
생성방법은 이미 만들어진 개인키에서 파생하도록 한다.

```bash
# 개인키(private.key)로부터 공개키(public.key)를 추출
openssl rsa -in private.key -pubout -out public.key
```

이러한 개인키와 공개키를 기반으로 암호화/복호화 과정을 거치게되는데, 이는 다음과 같다.

2. Client 는 본인의 Private Key 를 통해 수신 받은 데이터를 복호화한다.

이 암호화 과정을 기반으로 TLS 처리 과정을 거시적으로 보자면 다음과 같다.

2. 이 때 Client 의 6계층에서는 다음과 같은 과정이 일어난다.
3. Server → 7,6,5,/,,,,/1 → Client 를 거칠 때 7계층에서 넘어온 데이터를 6계층에서 암호화한다.

## TLS 처리과정

SSL : secure sockets layer
TLS : transport layer security
current “SSL” is actually TLS
What TLS does is 1) Authenticate the data 2) Encrypt it between User and Server
openssl command does making a SSL certificate (which is actually TSL certificate)
