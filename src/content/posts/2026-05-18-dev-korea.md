## non domain errors

## check exception does not scale

you can make method express checked exception on signature, but it does not scale -- multiple try-catches ,,,
you can leverage using sealed interface here with response wrapper and you can union these wrapper
you can improve this using Either interface like this

```kotlin
sealed interface LoginError {
    data object InvalidPassword : LoginError
    data object UserNotFound : LoginError
}

fun login(username: String, password: String): Either<LoginError, Session>
```
