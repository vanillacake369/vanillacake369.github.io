# Intro

알고리즘은 평생 숙제라고 생각한다.
어느 날 멈추면 컴퓨터답게 생각하는 방법과 그것에 맞게 코딩하는 그 머슬 메모리를 잃어버린다.
이 포스트에서는 책들과 문제들을 풀어보면서 개념과 테크닉을 다룬다.
주로 문제나 참고한 출처는 리트코드, 백준, 프로그래머스, [JusticeHui 님 블로그](https://justicehui.github.io/), `Competitive Programming 3: Steven Halim` 을 참고하였다.

# Thought Process & 시간복잡도

> Modern computers are quite fast and can process5 up to $\approx 100M (or 10^8; 1M = 1,000,000)$ operations in a few seconds. 
> You can use this information to determine if your algorithm will run in time. 
> *-Competitive Programming Ch 1.2.3 Tip3:Do Algorithm Analysis*
> 현대 컴퓨터는 매우 빠르며, 몇 초 만에 최대 $\approx 100M (or 10^8; 1M = 1,000,000)$회의 연산을 처리할 수 있다. 이 정보를 활용하여 알고리즘이 시간 내에 실행될지 판단할 수 있다.

문제를 풀려면 무작정 무지성 코딩을 할 것이 아니라 Thought Process 를 정리해야한다. 요컨데 문제의 조건을 분석하며 폭포수처럼 단계 별로 알고리즘과 관점을 생각해보는 것이다.
가령 특정 조건을 만족하도록 노드를 제거해야하는 문제라면 다음과 같이 분석할 수 있다.
```
완전탐색으로 노드를 제거할 수 있나? 
-(안된다면)-> 지울 바위 조합을 만들 수 있나?
-(안된다면)-> 이진탐색을 활용하여 조건에 만족하도록 노드를 제거할 수 있나?
```

또한 인지해야할 것은 시간복잡도이다. 앞서 봤듯 대부분의 코딩테스트 문제들은 $10^8 혹은 10^{10}$ 내면 풀린다.
대부분의 문제에서는 입력제한조건에는 출제자의 의도가 담겨있기 마련이다. 따라서 시간복잡도를 계산했을 때 초과한다면 틀린 접근이다.
가령 $10^{20}$ 에 해당하는 문제가 주어졌을 때는 이는 *$O(logN)$ 내로 풀어라* 를 시사한다.
따라서 이를 활용해 내가 생각하고 있는 알고리즘의 시간복잡도를 계산하고 아니라면 빠르게 다른 해석이나 알고리즘으로 넘어갈 수 있다.
이 때 만약 N 이 $10^{K}$ 꼴로 나오지 않는다면, $\mathbf{2^{10} \approx 10^3}$ 를 활용할 수 있다. -- 이 원리를 이용해서`2^10`을 1KB(대략 1,000)라고 부른다
    $$10^{18} = (10^3)^6 \approx (2^{10})^6 \approx 2^{60}$$

```java
static long[] prefixSum(int[] a) {
    long[] prefix = new long[a.length + 1];
    for (int i = 0; i < a.length; i++) {
        prefix[i + 1] = prefix[i] + a[i];

    return prefix;
}

static long rangeSum(long[] prefix, int l, int r) {
```

> 📓 이렇게도 활용할 수 있다! https://leetcode.com/problems/sum-of-distances/
>
> 특정값이 등장하는 인덱스 간의 절댓값 거리 합의 최적화 O(N^2) → O(N)
>
> Sum(i=1~length) |Pi - Pj|
>
> e.g. [1,2,4,8,10]
>
> - Pi < Pj -> (Pi \* leftCount) - leftSum
>   e.g. 4 의 왼쪽 절대거리 합 = (4-1) + (4-2) = (4\*2) - (1+2)
> - Pi > Pj -> RightSum - (Pi \* rightCount)
>   e.g. 4 의 오른쪽 절대거리 합 = (8-4) + (10-4) = (10+8) - (4\*2)
>
> 따라서 Pi 번 인덱스 절대거리합은
>
> pi \* (leftCount - rightCount) - leftsum + rightsum

```java
for (int val : count.keySet()) {
    List<Integer> indexes = count.get(val);
    int size = indexes.size();

    long[] prefixSum = new long[size + 1];
    for (int i = 0; i < size; i++) {
        prefixSum[i + 1] = prefixSum[i] + indexes.get(i);
    }

    for (int i = 0; i < size; i++) {
        int Pi = indexes.get(i);
        long leftSum = prefixSum[i];
        long rightSum = prefixSum[size] - prefixSum[i + 1];
        int leftCount = i;
        int rightCount = size - 1 - i;
        result[Pi] =
            (long) Pi * (leftCount - rightCount)
            - leftSum
            + rightSum;
    }
}

```

```java
// start부터 len만큼 원형으로 순회하며 방문 (MSB/LSB 같은 의미 X, 단순 인덱싱)
static void walkCircle(int n, int start, int len) {
    int idx = ((start % n) + n) % n; // 음수 start 방어
    for (int step = 0; step < len; step++) {
        // visit idx
        idx = (idx + 1) % n;
    }
}
```

### 문제 : [Shortest Distance to Target String in a Circular Array](https://leetcode.com/problems/shortest-distance-to-target-string-in-a-circular-array/)

> 요구사항
>
> - 원형 배열에서 `startIndex`에서 시작해 `target` 문자열에 도달하는 최소 이동 횟수

> 접근/원리
>
> - `target`이 있는 모든 인덱스 `i`에 대해 `min(|i-start|, n-|i-start|)` 최소값

```java
class Solution {
    public int closestTarget(String[] words, String target, int startIndex) {
        int n = words.length;
        int ans = Integer.MAX_VALUE;

        for (int i = 0; i < n; i++) {
            if (!words[i].equals(target)) continue;
            int d = Math.abs(i - startIndex);
            ans = Math.min(ans, Math.min(d, n - d));
        }

        return ans == Integer.MAX_VALUE ? -1 : ans;
    }
}
```

### 문제 : [Minimum Distance to the Target Element](https://leetcode.com/problems/minimum-distance-to-the-target-element/)

> 요구사항
>
> - 배열에서 값이 `target`인 인덱스들 중 `start`와의 최소 거리

> 접근/원리
>
> - 한 번 순회하며 `nums[i] == target`이면 `min(ans, |i-start|)`

```java
class Solution {
    public int getMinDistance(int[] nums, int target, int start) {
        int ans = Integer.MAX_VALUE;
        for (int i = 0; i < nums.length; i++) {
            if (nums[i] == target) {
                ans = Math.min(ans, Math.abs(i - start));
            }
        }
        return ans;
    }
}
```

### 문제 : [Two Furthest Houses With Different Colors](https://leetcode.com/problems/two-furthest-houses-with-different-colors/)

> 요구사항
>
> - 서로 다른 색을 가진 두 집의 최대 거리(인덱스 차이)

> 접근/원리
>
> - 끝점(0, n-1)을 기준으로 반대쪽에서 “처음으로 다른 색”이 나오는 지점만 보면 충분

```java
class Solution {
    public int maxDistance(int[] colors) {
        int n = colors.length;
        int ans = 0;

        for (int i = 0; i < n; i++) {
            if (colors[i] != colors[n - 1]) {
                ans = Math.max(ans, (n - 1) - i);
                break;
            }
        }
        for (int i = n - 1; i >= 0; i--) {
            if (colors[i] != colors[0]) {
                ans = Math.max(ans, i);
                break;
            }
        }

        return ans;
    }
}
```

### 문제 : 방 배정

https://www.acmicpc.net/problem/13300

> 💡
>
> A 를 B 로 나눴을 때 나머지가 0보다 크면 몫 + 1 처리
>
> → ( A + B - 1 ) / B
>
> - 설명
>   A = B \* q + r 이다. (q : 몫, r : 나머지, 0 ≤ r < B)
>   (A + B - 1) / B 는 위 공식에 따라 ( B \* q + r - 1 ) / B 가 된다.
>   이는 곧 (q + 1) + (r - 1)/B 가 된다.
>   만약 A 가 B 배수가 되어 r =0 이 되면 위 식은 q + 1 + (-1)/B = q + 1 -1 = q 가 된다.
>   만약 A 가 B 배수가 아닌 경우 0 ≤ r < B 에서 0 ≤ r - 1 < B 가 성립되므로 (r-1)/B 는 0이 된다. 즉, (q + 1) + (r - 1)/B = q + 1 이 된다.

```java
private static int getRoomCount(int[][] studentCounts, int k) {
    int count = 0;

    for (int i = 0; i < studentCounts.length; i++) {
        count += studentCounts[i][0] / 2 + studentCounts[i][0] % 2;
    }

    return count;
}
```

### 문제 : Closest Equal Element Queries

> 요구사항
>
> - 원형 배열 `nums`에서 각 `queries[i]` 위치의 값과 같은 값을 갖는 “다른 인덱스”까지의 최소 이동 거리

> 접근/원리
>
> - 값별 인덱스 리스트를 만들고, 쿼리 인덱스의 prev/next만 원형 거리로 비교

```java
import java.util.*;

class Solution {
    public List<Integer> solveQueries(int[] nums, int[] queries) {
        int n = nums.length;
        Map<Integer, List<Integer>> pos = new HashMap<>();
        for (int i = 0; i < n; i++) {
            pos.computeIfAbsent(nums[i], k -> new ArrayList<>()).add(i);
        }

        List<Integer> ans = new ArrayList<>(queries.length);
        for (int q : queries) {
            List<Integer> idxs = pos.get(nums[q]);
            if (idxs.size() == 1) {
                ans.add(-1);
                continue;
            }

            int p = Collections.binarySearch(idxs, q);
            int prev = idxs.get((p - 1 + idxs.size()) % idxs.size());
            int next = idxs.get((p + 1) % idxs.size());

            int d1 = circularDist(q, prev, n);
            int d2 = circularDist(q, next, n);
            ans.add(Math.min(d1, d2));
        }
        return ans;
    }

    private int circularDist(int i, int j, int n) {
        int d = Math.abs(i - j);
        return Math.min(d, n - d);
    }
}
```

### 문제 : Minimum Absolute Distance Between Mirror Pairs

> 요구사항
>
> - `nums[i]`와 `reverse(nums[j])`가 같아지는 쌍들 중, `|i-j|`의 최소값

> 접근/원리
>
> - 순회하면서 현재 `nums[i]`가 map에 있으면 거리 갱신, 이후 `reverse(nums[i])`를 map에 저장

```java
import java.util.*;

class Solution {
    public int minMirrorPairDistance(int[] nums) {
        int ans = Integer.MAX_VALUE;
        Map<Integer, Integer> seen = new HashMap<>(); // value -> index

        for (int i = 0; i < nums.length; i++) {
            Integer j = seen.get(nums[i]);
            if (j != null) ans = Math.min(ans, i - j);
            seen.put(reverseInt(nums[i]), i);
        }

        return ans == Integer.MAX_VALUE ? -1 : ans;
    }

    private int reverseInt(int x) {
        long r = 0;
        while (x != 0) {
            r = r * 10 + (x % 10);
            x /= 10;
        }
        return (int) r;
    }
}
```

### 테크닉 : **Monotonic Stack, 순서 유지하면서 다음 큰 원소 찾기**

https://justicehui.github.io/medium-algorithm/2019/01/01/monotoneStack/

> 💡 핵심 원리
>
> - ‘배열의 순서를 유지하면서 자신보다 크고 작은 수의 위치/개수 판별’ 할 때 사용
> - 스택에 활용하여**x 이상의 수를 모두 제거하고** x를 넣어서 아래 상태를 처리
>   ![](https://i.imgur.com/LwF5sPx.png)
> - 이를 통해 Monotonic (단조, 오름/내림) 을 유지하며
>   x 에 대해 왼쪽/오른쪽에서 가장 큰수, 혹은 x 보다 작은 갯수 등등을 구할 수 있음
> - `while(pop 조건)`으로 정리하면서 진행
>     - `pop 조건` → 1) stack 이 비어있지 않으면서 AND 2) stack 에 집어넣은 수가 배열의 수에 대해 조건을 만족할 때
>     - `pop`되는 조건을 만족한 순간이 “정답이 확정되는 타이밍”
>       (오큰수/다음 큰 원소/가장 가까운 큰 원소)

> Tip 💡
>
> - “왼쪽에서 가까운 큰 원소”는 진행 방향을 바꾸거나(오→왼) `while` 조건을 반대로 잡으면 된다

```java
static int[] nextGreater(int[] a) {
    int n = a.length;
    int[] ans = new int[n];
    Arrays.fill(ans, -1);
    Deque<Integer> st = new ArrayDeque<>();

    for (int i = 0; i < n; i++) {
        while (!st.isEmpty() && a[st.peek()] < a[i]) {
            ans[st.pop()] = a[i];
        }
        st.push(i);
    }
    return ans;
}
```

### 문제 : [오아시스 재결합](https://www.acmicpc.net/problem/3015)

```java

/**
 * # 해석
 * 두 위치의 수 사이에 대해 조건에 맞는지 판별
 *
 * # 접근
 * 루프를 통해 접근 시 N 이 500,000 이므로 시간복잡도 초과
 * 따라서 LIFO 기반 수를 순서대로 입력하고
 * 내가 누군가에게 보여질 수 있는지 조건을 판별
 *
 * # 상태값
 * 이전의 수들의 볼 수 있는 상태를 연속해서 가져가야하므로
 * {수의 크기, 볼 수 있는 횟수} 를 가져가서
 * 이전 수들의 볼 수 있는 횟수를 캐싱
 */
private static long resolve(int n, int[] inputs) {
    long sum = 0;

    Deque<int[]> stack = new ArrayDeque<>();

    for (int i = 0; i < n; i++) {
        int currentCache = 1;
        while (!stack.isEmpty()) {
            int[] prevPerson = stack.peek();
            if (prevPerson[0] < inputs[i]) {
                sum += prevPerson[1];
                stack.pop();
                continue;
            }
            if (prevPerson[0] == inputs[i]) {
                sum += prevPerson[1];
                currentCache = prevPerson[1] + 1;
                stack.pop();
                continue;
            }
            sum++;
            break;
        }
        stack.push(new int[] {inputs[i], currentCache});
    }

    return sum;
}

```

```java
// 상태 역추적
// BASE CASE : 가장 최상위 부모노드 -- row 가 1 이거나, 노드가 하나
// 다음 부모위치를 계산하여 부모 상태에 대한 재귀 호출
// 재귀호출하여 구한 부모의 값을 통해 주어진 패턴에 따라 현재 값을 결정
static int solve(int n, long k) {
    if (n == 1) return 0;

    long parentK = (k + 1) / 2;

    int parentValue = solve(n - 1, parentK);

    return (k % 2 == 0) ? invert(parentValue) : parentValue;
}
```

```java
static void quad(int[][] a, int r, int c, int size, int[] count01) {
    int first = a[r][c];
    boolean uniform = true;
    for (int i = r; i < r + size && uniform; i++) {
        for (int j = c; j < c + size; j++) {
            if (a[i][j] != first) {
                uniform = false;
                break;
            }
        }
    }

    if (uniform) {
        count01[first]++;
        return;

    int half = size / 2;
    quad(a, r, c, half, count01);
    quad(a, r, c + half, half, count01);
    quad(a, r + half, c, half, count01);
    quad(a, r + half, c + half, half, count01);
}
```

### 문제 : [K-th Symbols](https://leetcode.com/problems/k-th-symbol-in-grammar/description/)

> 요구사항
>
> - 첫 번째 행을 `0`으로 시작하여, 각 `0`은 `01`로, `1`은 `10`으로 치환하며 n번째 행까지 확장한다.
> - n번째 행의 k번째(1-indexed) 숫자를 반환하라.
> - n 의 최대가 30

> 접근/원리
>
> - n 의 최대가 30 → 시간초과범위인 10! 을 넘어감, 따라서 전체 수열 생성 불가
> - 상태 역추적 테크닉을 사용 — k번째 노드가 부모노드의 어떤 값으로부터 유도되었는지 역추적
>     - k 가 홀수(왼쪽 자식) → 부모의 값을 **그대로**
>     - k 가 짝수(오른쪽 자식) → 부모의 값을 **반전**

```java
class Solution {
    public int kthGrammar(int n, int k) {
        // Base Case: 1행은 무조건 0
        if (n == 1) {
            return 0;
        }

        // 1. 부모 인덱스 계산 (1-indexed 기준)
        int parentK = (k + 1) / 2;

        // 2. 부모의 값을 재귀적으로 알아냄
        int parentValue = kthGrammar(n - 1, parentK);

        // 3. 현재 위치(k)가 부모의 왼쪽(홀수)인지 오른쪽(짝수)인지에 따라 결정
        if (k % 2 != 0) {
            // 왼쪽 자식: 부모 값 그대로
            return parentValue;
        } else {
            // 오른쪽 자식: 부모 값 반전 (0->1, 1->0)
            return parentValue == 0 ? 1 : 0;
        }
    }
}
```

## BFS

각 노드들의 이웃노드들을 순차적으로 탐색해나가는 방식 cf) DFS 는 노드의 자식의 자식을 재귀호출을 통해 탐색

Queue 를 사용해서 다음 방문할 곳을 순서대로 저장

Visited 를 통해 방문여부를 표시하여 노드들을 탐색

- 시작 지점을 **Queue**에 넣고 **방문 처리**를 한다.
- 큐가 빌 때까지 다음을 반복한다:
    - 큐에서 하나를 꺼낸다 (`poll`).
    - 그 주변(상하좌우)을 살핀다.
    - **처음 가보는 곳**이고 **갈 수 있는 길**이라면? -> 큐에 넣고 방문 처리를 한다.

```java
// BFS에 사용할 큐를 생성해줍니다.
int[][] graph = {{}, {2,3,7}, {1,3,5}, {1,2}, {6,8}, {2}, {4,7,8}, {1,6}, {4,6}};
boolean[] visit = new boolean[9];
Queue<Integer> q = new LinkedList<Integer>();

// 큐에 BFS를 시작 할 노드 번호를 넣어줍니다.
q.offer(start);

// 시작노드 방문처리
visited[start] = true;

// 큐가 빌 때까지 반복
while(!q.isEmpty()) {
	int nodeIndex = q.poll();

	//큐에서 꺼낸 노드와 연결된 노드들 체크
	for(int i=0; i<graph[nodeIndex].length; i++) {
		int temp = graph[nodeIndex][i];
		// 방문하지 않았으면 방문처리 후 큐에 넣기
		if(!visited[temp]) {
			visited[temp] = true;
			q.offer(temp);
		}
	}
}
```

### 테크닉 : State Space BFS(상태 사용하여 조건에 따른 탐색)

> 💡 핵심 원리
>
> - 단순히 좌표만 탐색하는 것이 아니라, '현재 어떤 상태인가' 를 좌표의 확장된 축으로 보고 탐색
>   → 특정 아이템 획득(열쇠), 특수 능력 사용(신발/벽 부수기), 등 **조건부 최단 거리**를 구할 때 사용
> - **방문 배열의 확장:** `visited[x][y]` → `visited[x][y][state]`
>     - `state`는 보통 정수(0, 1)나 비트마스킹(Bitmask)으로 관리
>     - 같은 좌표라도 `state`가 다르면 '완전히 다른 지점' 으로 간주하여 독립적으로 처리

```java
static int bfs(int n, int m, int[][] map) {
		// 방문여부
    int[][][] visited = new int[n][m][2];
    for (int[][] row : visited){
		    for (int[] col : row) {
				    Arrays.fill(col, -1);
		    }
    }

		// 일반이동과 특수이동에 대해 BFS 진행
		// 각 노드에 대해 특수이동한 경우의 수들로 진행 -- 조합처럼
    Queue<int[]> q = new LinkedList<>();
    q.add(new int[]{0, 0, 0});
    visited[0][0][0] = 0;

    while (!q.isEmpty()) {
        int[] curr = q.poll();
        int r = curr[0];
        int c = curr[1];
        int used = curr[2];

        if (r == n - 1 && c == m - 1){
		        return visited[r][c][used];
        }

        for (int i = 0; i < 4; i++) {
            // 1. 일반 이동 (항상 가능)
            int nr = r + dr[i], nc = c + dc[i];
            if (canGo(nr, nc) && visited[nr][nc][used] == -1) {
                visited[nr][nc][used] = visited[r][c][used] + 1;
                q.add(new int[]{nr, nc, used});
            }

            // 2. 특수 이동 (신발 미사용 시에만 가능)
            if (used == 0) {
                int sr = r + dr[i] * 2, sc = c + dc[i] * 2;
                if (canGo(sr, sc) && visited[sr][sc][1] == -1) {
                    visited[sr][sc][1] = visited[r][c][0] + 1;
                    q.add(new int[]{sr, sc, 1});
                }
            }
        }
    }
    return -1;
}
```

### 문제 : [보물 지도](https://school.programmers.co.kr/learn/courses/15009/lessons/121690)

> 요구사항
>
> - **목표:** (1, 1)에서 출발하여 함정을 피해 (n, m)에 도착하는 **최단 시간** 계산
> - **이동 규칙:**
>     - **일반 걷기:** 상하좌우 1칸 이동 (시간 1 소요)
>     - **신비로운 신발:** 상하좌우 방향으로 **2칸 점프** (시간 1 소요, 함정 뛰어넘기 가능)
> - **제약 조건:** 신발은 전체 경로 중 **딱 1번**만 사용 가능
> - **예외 처리:** 보물 지점에 도달 불가능한 경우 `1` 반환

> 접근/원리
>
> - 큐를 통해 각 노드들의 인접노드들을 BFS 순회하며 방문여부 및 방문거리 체크 — 토마토 문제처럼
> - 다만 이 때 (x,y) 가 아닌 (x,y,used) 축을 추가하여 신발 사용 여부 추가
>     - 이를 통해 (1,1,신발X), (1,1,신발O) 에 대해 각각 BFS 를 진행할 수 있음
>     - 분기에 따라 BFS 경로 중 가장 빨리 마지막 노드에 도달하는 게 곧 최단시간

```java
import java.util.*;

class Solution {
    private int N, M;
    private boolean[][] isHole;
    private int[][][] visited;
    private final int[] dx = {0, 0, 1, -1};
    private final int[] dy = {1, -1, 0, 0};

    public int solution(int n, int m, int[][] hole) {
        // 1. 초기 설정 및 맵 구성
        setup(n, m, hole);

        // 2. 탐색 시작 (BFS)
        return findShortestPath();
    }

    private void setup(int n, int m, int[][] hole) {
        this.N = n;
        this.M = m;
        this.isHole = new boolean[n + 1][m + 1];
        for (int[] h : hole) {
            isHole[h[0]][h[1]] = true;
        }

        this.visited = new int[n + 1][m + 1][2];
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= m; j++) {
                Arrays.fill(visited[i][j], -1);
            }
        }
    }

    private int findShortestPath() {
        Queue<int[]> q = new LinkedList<>();
        q.add(new int[]{1, 1, 0}); // {x, y, usedShoes}
        visited[1][1][0] = 0;

        while (!q.isEmpty()) {
            int[] curr = q.poll();
            int x = curr[0];
            int y = curr[1];
            int used = curr[2];

            // 목표 지점 도달 시 즉시 반환
            if (x == N && y == M) return visited[x][y][used];

            // 네 방향으로 이동 시도
            for (int i = 0; i < 4; i++) {
                handleNormalMove(q, x, y, used, i);
                handleShoeMove(q, x, y, used, i);
            }
        }
        return -1;
    }

    private void handleNormalMove(Queue<int[]> q, int x, int y, int used, int dir) {
        int nx = x + dx[dir];
        int ny = y + dy[dir];

        if (canMoveTo(nx, ny) && visited[nx][ny][used] == -1) {
            visited[nx][ny][used] = visited[x][y][used] + 1;
            q.add(new int[]{nx, ny, used});
        }
    }

    private void handleShoeMove(Queue<int[]> q, int x, int y, int used, int dir) {
        // 신발을 이미 썼다면 무시
        if (used == 1) return;

        int nx = x + dx[dir] * 2;
        int ny = y + dy[dir] * 2;

        // 신발 사용 후 상태는 무조건 1(used)
        if (canMoveTo(nx, ny) && visited[nx][ny][1] == -1) {
            visited[nx][ny][1] = visited[x][y][0] + 1;
            q.add(new int[]{nx, ny, 1});
        }
    }
```

### 문제 : [그림](https://www.acmicpc.net/problem/1926)

```java
public int[] solution {int n, int m, int[][] paint) {
	Queue<int[]> q = new LinkedList<>();
	int[] dr = {-1,1,0,0};
	int[] dc = {0,0,-1,1};
	boolean[][] visited = new boolean[n][m];
	int count = 0;
	int maxSize = 0;

	for (int r=0;r<n;r++){
		for(int c=0;c<m;c++){
			// 현재 노드가 그림인 경우
			// 현재 노드 방문 처리 && 이웃노드들에 대한 BFS 시작
			if ( isPaint(paint,r,c) && visited[r][c] == false ) {
				visited[r][c] = true;
				q.offer(new int[]{r,c});
				count++;
				int currentSize =0;

				while ( !q.isEmpty() ){
					int[] currNode = q.poll();
					int cr = currNode[0];
          int cc = currNode[1];
					currentSize ++;

					for (int i=0;i<4;i++){
						nr = cr + dr[i];
						nc = cc + dc[i];
						// 다음 노드가 범위 내 그림이고 미방문이라면
						// 방문처리하고 방문대기큐에 추가
						if ( isRange(paint,nr,nc) && isPaint(paint,nr,nc) && visited[nr][nc] == false ){
							visited[nr][nc] = true;
							q.offer(new int[]{nr,nc});
						}
					}
				}

				maxSize = Math.max(maxSize,currentSize);
			}
		}
	}

	return new int[]{count, maxSize);
}

}
```

## DFS/백트래킹

한 방향으로 갈 수 있을 때까지 깊게 탐색하고, 더 이상 갈 곳이 없으면 가장 마지막 갈림길로 돌아와 다른 방향을 탐색하는 방식

- 사용 자료구조: 시스템 Stack(재귀 호출) 또는 명시적 `Stack` 클래스 사용.
- 탐색 메커니즘: 현재 노드와 연결된 인접 노드 중 하나를 선택해 즉시 깊게 들어간다. (자식의 자식을 먼저 방문)
- 백트래킹(Backtracking): 막다른 길에 도달하면 호출 스택을 빠져나와(Return) 이전 상태로 복귀한다.
- 방문 처리(Visited): 무한 루프 방지를 위해 필수이며, 문제 유형에 따라 복귀 시 방문 처리를 해제하기도 한다.

```java
boolean[] visited = new boolean[9];

// DFS 실행 함수
public void dfs(int nodeIndex) {
    // 1. 현재 노드 방문 처리
    visited[nodeIndex] = true;
    System.out.print(nodeIndex + " -> "); // 탐색 순서 확인용

    // 2. 인접한 노드들을 확인
    for (int nextNode : graph[nodeIndex]) {
        // 3. 방문하지 않은 노드가 있다면 즉시 깊게 탐색(재귀 호출)
        if (!visited[nextNode]) {
            dfs(nextNode);
        }
    }
}
```

>

> 접근/원리
>

```java
import java.util.*;

class Solution {
    public List<List<Integer>> subsets(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        backtrack(0, nums, new ArrayList<>(), result);
        return result;
    }

    private void backtrack(int start, int[] nums, List<Integer> temp, List<List<Integer>> result) {
        result.add(new ArrayList<>(temp));
        for (int i = start; i < nums.length; i++) {
            temp.add(nums[i]);
            backtrack(i + 1, nums, temp, result);
            temp.remove(temp.size() - 1);
        }
    }
}
```

> 요구사항
>
> - 배열 `nums`의 모든 순열을 반환

> 접근/원리
>
> - `visited[i]`로 사용 여부를 관리하며 길이 `n`이 되면 스냅샷 저장

```java
import java.util.*;

class Solution {
    public List<List<Integer>> permute(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        backtrack(nums, new boolean[nums.length], new ArrayList<>(), result);
        return result;
    }

    private void backtrack(int[] nums, boolean[] visited, List<Integer> temp, List<List<Integer>> result) {
        if (temp.size() == nums.length) {
            result.add(new ArrayList<>(temp));
            return;
        }

        for (int i = 0; i < nums.length; i++) {
            if (visited[i]) continue;
            visited[i] = true;
            temp.add(nums[i]);
            backtrack(nums, visited, temp, result);
            temp.remove(temp.size() - 1);
            visited[i] = false;
        }
    }
}
```

> 요구사항
>
> - `1..n`에서 `k`개를 뽑는 모든 조합을 반환

> 접근/원리
>
> - `start`부터 다음 수를 선택하며 내려가고, 크기 `k`가 되면 스냅샷 저장

```java
import java.util.*;

class Solution {
    public List<List<Integer>> combine(int n, int k) {
        List<List<Integer>> result = new ArrayList<>();
        backtrack(1, n, k, new ArrayList<>(), result);
        return result;
    }

    private void backtrack(int start, int n, int k, List<Integer> cur, List<List<Integer>> result) {
        if (cur.size() == k) {
            result.add(new ArrayList<>(cur));
            return;
        }

        for (int i = start; i <= n; i++) {
            cur.add(i);
            backtrack(i + 1, n, k, cur, result);
            cur.remove(cur.size() - 1);
        }
    }
}
```

> 💡 한 줄 정의
>
> - “서로 연결됐는지(같은 컴포넌트인지)”를 빠르게 묻고 합치는 자료구조
> - `init`(초기화) + `find`(대표 찾기) + `union`(합치기) 의 함수를 통해
>   서로소 집합인지 판단
> - 배열 구현방식/트리 구현방식이 존재
>   배열의 경우 find:o(1), union:o(n) 이므로
>   대부분의 경우 트리 — parent[i] = i번 노드의 부모 노드 — 방식을 사용
>   트리의 find:O(n) 을 극복하기 위해 경로압축 & rank 로 최적화하여 사용

### **테크닉 : Union-Find / 서로소 집합**

![](https://i.imgur.com/V0nRfua.png)

```java
static class DSU {
    int[] parent;
    int[] size;

    DSU(int n) {
        parent = new int[n];
        size = new int[n];
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            size[i] = 1;
        }
    }

		// 'X 에 해당하는 부모노드 찾기'
		//
		// '경로 압축' 최적화 :
		// Find 연산을 수행할 때 거쳐가는 모든 노드를 루트 노드 바로 밑에 붙여
		// 부모노드를 찾는 재귀호출 수를 줄여 최적화
    int find(int x) {
        if (parent[x] == x) return x;
        parent[x] = find(parent[x]);
        return parent[x];
    }

		// 'a 노드의 트리에 b 노드의 트리를 병합'
		//
		// 'Rank' 최적화 :
		// 높이가 더 낮은 트리를 높은 트리 밑에 넣어
		// Unbalanced Tree 를 지양하여 최적화
    boolean union(int a, int b) {
        int ra = find(a), rb = find(b);
        if (ra == rb) return false;
        if (size[ra] < size[rb]) {
		        // (a <- b 병합이므로 b 가 크다면 스왑)
            int tmp = ra; ra = rb; rb = tmp;
        }
        parent[rb] = ra;
        size[ra] += size[rb];
        return true;
    }
}
```

### 문제 : [Minimize Hamming Distance After Swap Operations](https://leetcode.com/problems/minimize-hamming-distance-after-swap-operations/?envType=daily-question&envId=2026-04-21)

> 요구사항
>
> - 배열 `source`, `target`이 있고, `allowedSwaps`로 연결된 인덱스끼리는 자유롭게 swap 가능
> - swap을 적절히 해서 Hamming distance(서로 다른 위치 개수)를 최소화

> 접근/원리
>
> - 인덱스 0번과 1번을 바꿀 수 있고, 1번과 2번을 바꿀 수 있으므로 바꿀 수 있는 그룹을 Union Find 구조로 처리
> - 처리한 Union Find 구조체에서 각 컴포넌트별로 `source` 값 빈도표를 만들고, `target` 값을 소비하면서 mismatch만 카운트

```java
import java.util.*;

class Solution {
    public int minimumHammingDistance(int[] source, int[] target, int[][] allowedSwaps) {
        int n = source.length;
        DSU dsu = new DSU(n);
        for (int[] e : allowedSwaps) dsu.union(e[0], e[1]);

        Map<Integer, Map<Integer, Integer>> bagByRoot = new HashMap<>();
        for (int i = 0; i < n; i++) {
            int root = dsu.find(i);
            bagByRoot.computeIfAbsent(root, k -> new HashMap<>())
                     .merge(source[i], 1, Integer::sum);
        }

        int diff = 0;
        for (int i = 0; i < n; i++) {
            int root = dsu.find(i);
            Map<Integer, Integer> bag = bagByRoot.get(root);
            int v = target[i];
            int cnt = bag.getOrDefault(v, 0);
            if (cnt == 0) diff++;
            else if (cnt == 1) bag.remove(v);
            else bag.put(v, cnt - 1);
        }
        return diff;
    }

    static class DSU {
        int[] parent;
        int[] size;

        DSU(int n) {
            parent = new int[n];
            size = new int[n];
            for (int i = 0; i < n; i++) {
                parent[i] = i;
                size[i] = 1;
            }
        }

        int find(int x) {
            if (parent[x] == x) return x;
            parent[x] = find(parent[x]);
            return parent[x];
        }

        void union(int a, int b) {
            int ra = find(a), rb = find(b);
            if (ra == rb) return;
            if (size[ra] < size[rb]) {
                int tmp = ra; ra = rb; rb = tmp;
            }
            parent[rb] = ra;
            size[ra] += size[rb];
        }
    }
}
```

### 문제 : [Count Complete Tree Nodes](https://leetcode.com/problems/count-complete-tree-nodes/submissions/1994814745/)

> 요구사항
>
> - 완전 이진 트리(Complete Binary Tree)의 전체 노드 개수 계산하되
>   O(N) 보다 빠른 시간 복잡도로 구현

> 접근/원리
>
> - 포화 이진 트리인지 확인하여 포화 이진트리의 특성을 활용
>     - 왼쪽 끝 높이와 오른쪽 끝 높이를 각각 연산
>     - 서로 같다면, 해당 트리는 꽉 찬 상태이므로 노드 개수는 2^height - 1 임
> - 분할 정복
>     1. 왼쪽 끝과 오른쪽 끝 높이를 측정한다.
>     2. 같다면 공식을 써서 즉시 반환
>     3. 다르다면 `1 + countNodes(left) + countNodes(right)`로 재귀 탐색.

```java
class Solution {
    public int countNodes(TreeNode root) {
        if (root == null) return 0;

        int leftH = getLeftHeight(root);
        int rightH = getRightHeight(root);

        // 1. 성질 이용: 왼쪽/오른쪽 높이가 같으면 포화 이진 트리임
        if (leftH == rightH) {
            // (1 << leftH)는 2^leftH와 같음
            return (1 << leftH) - 1;
        }

        // 2. 높이가 다를 때만 자식으로 내려감 (이 중 한쪽은 반드시 포화 트리)
        return 1 + countNodes(root.left) + countNodes(root.right);
    }

    private int getLeftHeight(TreeNode node) {
        int h = 0;
        while (node != null) {
            h++;
            node = node.left;
        }
        return h;
    }

    private int getRightHeight(TreeNode node) {
        int h = 0;
        while (node != null) {
            h++;
            node = node.right;
        }
        return h;
    }
}
```

## 문제 : [Longest Common Suffix Queries](https://leetcode.com/problems/longest-common-suffix-queries/)

> 요구사항
> - 두 문자열 배열 `wordsContainer`와 `wordsQuery`가 주어짐
> - 각 Query 단어에 대해, wordsContainer에서 가장 긴 공통 접미사(Longest Common Suffix)를 가지는 단어의 인덱스를 찾아야 함

	이마저도 같다면 2) Container에서 가장 먼저 등장한(인덱스가 작은) 단어

> 접근/원리
> - 공통 접미사(Suffix) 문제를 해결하기 위해, 모든 단어를 **뒤집어서(Reverse)** 트라이(Trie)에 삽입함으로써 **공통 접두사(Prefix)** 문제로 치환
> - **트라이 노드 최적화 (실시간 마킹):** 단어를 뒤집어 트라이에 `insert`하는 과정에서, 거쳐 가는 모든 노드에 "현재 노드를 공유하는 단어들 중 최적의 단어 조건(최소 길이, 최소 인덱스)"을 실시간으로 갱신
    - 단어의 원본 길이가 더 짧다면 `node.idx`를 갱신합니다.
    - `wordsContainer`를 0번 인덱스부터 순서대로 삽입하므로, 길이가 같을 때는 자연스럽게 먼저 들어온 단어(최소 인덱스)가 유지됩니다.
> - **상수 시간 쿼리 ($O(L)$):** Query 단어를 뒤집어 트라이를 타고 내려가다가 매칭이 끊기거나 단어가 끝나면, 그 자리에 적혀있는 `node.idx`를 즉시 반환하여 정답 배열을 구축합니다.

```java
import java.util.*;

class TrieNode {
    TrieNode[] children = new TrieNode[26];
    int minLen = Integer.MAX_VALUE;
    int idx = Integer.MAX_VALUE;

    TrieNode() {
        for (int i = 0; i < 26; i++) {
            children[i] = null;
        }
    }
}

class Trie {
    TrieNode root = new TrieNode();

    // 단어를 역순으로 뒤집어 삽입하며 노드마다 최적의 인덱스를 실시간 마킹
    void insert(String s, int idx) {
        int len = s.length();
        TrieNode node = root;

        // 공통 접미사가 전혀 없는 경우(루트 노드)를 위한 기본값 세팅
        if (len < node.minLen) {
            node.minLen = len;
            node.idx = idx;
        }

        for (char ch : s.toCharArray()) {
            int c = ch - 'a';
            if (node.children[c] == null) {
                node.children[c] = new TrieNode();
            }
            node = node.children[c];

            // 타이브레이커 반영: 더 짧은 단어가 등장했다면 인덱스 갱신
            // (순차 삽입이므로 길이가 같을 때는 자동 생략되어 최소 인덱스가 유지됨)
            if (len < node.minLen) {
                node.minLen = len;
                node.idx = idx;
            }
        }
    }

    // 쿼리 단어를 역순으로 추적하며 가장 깊게 일치하는 지점의 최적 인덱스 반환
    int query(String s) {
        TrieNode node = root;

        for (char ch : s.toCharArray()) {
            int c = ch - 'a';
            if (node.children[c] != null) {
                node = node.children[c];
            } else {
                break; // 매칭이 끊기면 지금까지 내려온 최선의 정답 노드에서 중단
            }
        }

        return node.idx;
    }
}

class Solution {
	/**
	각 query arr 에 대한 lcs(longest common suffix) 찾고 조건에 맞춰 index 찾기
	
	lcs 를 찾고
	각 nominee 들에 대한 [nominee 사이즈, lcs 등장 index] 를 구하고
	이를 통해 조건에 따른 nominee 의 index 를 담은 answer arr 반환
	
	시뮬레이션은 안 되고 -> query 10^4 * container 10^3 이면 10^7 을 이미 넘어감
	그리디 문제도 아니고 안 됨 -> 매칭을 분기하는 그런 구조의 최적의 판단 문제 X
	맵이나 해시 가능??? 안 됨 -> 모든 단어의 suffix 를 쪼개서 넣을 수 있음, 다만 중복된 suffix 에 대한 메타데이터를 관리해야함
	점화식 가능??? 안 됨 -> 각 문자열 간 관계가 없어 수식 X
	스택이나 덱 가능??? 안 됨 -> 슬라이딩 쿼리의 문제가 X
	union find 가능??? 안 됨 -> 그룹 병합 문제가 X
	parametric search 가능??? 안 됨 -> monotone 의 구조가 아니기에 left,right 판단 기준 X
	문자열을 뒤집어서 Suffix 가 아닌 Prefix 를 찾는 Trie 자료구조 문제로 접근
**/
    public int[] stringIndices(String[] wordsContainer, String[] wordsQuery) {
        Trie trie = new Trie();

        // 1. 모든 Container 단어를 뒤집어서 트라이에 구축
        for (int i = 0; i < wordsContainer.length; i++) {
            String reversed = new StringBuilder(wordsContainer[i])
                .reverse()
                .toString();
            trie.insert(reversed, i);
        }

        // 2. 모든 Query 단어를 뒤집어서 트라이 검색 후 결과 수집
        int[] res = new int[wordsQuery.length];
        for (int i = 0; i < wordsQuery.length; i++) {
            String query = wordsQuery[i];
            String reversed = new StringBuilder(query).reverse().toString();
            res[i] = trie.query(reversed);
        }

        return res;
    }
}
```

```java
import java.util.*;
import java.io.*;

public class Main {
    static int N, M, K;
    static int[][] board;

    static class Sticker {
        int r, c;
        List<int[]> points; // 1이 적힌 칸의 상대 좌표 리스트

        Sticker(int r, int c, int[][] shape) {
            this.r = r;
            this.c = c;
            this.points = new ArrayList<>();
            for (int i = 0; i < r; i++) {
                for (int j = 0; j < c; j++) {
                    if (shape[i][j] == 1) {
                        points.add(new int[]{i, j});
                    }
                }
            }
        }

        // 스티커 90도 회전: (r, c) -> (c, r)
        // 좌표 변환: (oldR, oldC) -> (oldC, (maxR - 1) - oldR)
        void rotate() {
            List<int[]> nextPoints = new ArrayList<>();
            for (int[] p : points) {
                // 회전 후 새로운 행 인덱스는 기존의 열 인덱스
                // 회전 후 새로운 열 인덱스는 (기존 최대 행 - 1) - 기존 행 인덱스
                nextPoints.add(new int[]{p[1], (r - 1) - p[0]});
            }
            this.points = nextPoints;
            // 행과 열 크기 교체
            int temp = r;
            r = c;
            c = temp;
        }
    }

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        N = Integer.parseInt(st.nextToken());
        M = Integer.parseInt(st.nextToken());
        K = Integer.parseInt(st.nextToken());

        board = new int[N][M];

        for (int k = 0; k < K; k++) {
            st = new StringTokenizer(br.readLine());
            int r = Integer.parseInt(st.nextToken());
            int c = Integer.parseInt(st.nextToken());
            int[][] shape = new int[r][c];

            for (int i = 0; i < r; i++) {
                st = new StringTokenizer(br.readLine());
                for (int j = 0; j < c; j++) {
                    shape[i][j] = Integer.parseInt(st.nextToken());
                }
            }

            Sticker sticker = new Sticker(r, c, shape);
            solve(sticker);
        }

        System.out.println(countFilledCells());
    }

    static void solve(Sticker s) {
        // 0도, 90도, 180도, 270도 총 4번의 방향 확인
        for (int rot = 0; rot < 4; rot++) {
            // 보드의 (0,0)부터 순회하며 붙일 수 있는지 확인 (문제 조건: 위쪽 -> 왼쪽 순)
            for (int i = 0; i <= N - s.r; i++) {
                for (int j = 0; j <= M - s.c; j++) {
                    if (canAttach(s, i, j)) {
                        attach(s, i, j);
                        return; // 붙였으면 다음 스티커로 넘어감
                    }
                }
            }
            s.rotate(); // 못 붙였다면 90도 회전
        }
    }

    static boolean canAttach(Sticker s, int sy, int sx) {
        for (int[] p : s.points) {
            // 보드 범위를 벗어나거나 이미 다른 스티커가 붙어있는 경우
            if (board[sy + p[0]][sx + p[1]] == 1) return false;
        }
        return true;
    }

    static void attach(Sticker s, int sy, int sx) {
        for (int[] p : s.points) {
            board[sy + p[0]][sx + p[1]] = 1;
        }
    }

    static int countFilledCells() {
        int count = 0;
        for (int i = 0; i < N; i++) {
            for (int j = 0; j < M; j++) {
                if (board[i][j] == 1) count++;
            }
        }
        return count;
    }
}
```

```java
import java.util.*;

public class Main {
    static int maxScore = 0;
    // 방향: 1부터 8까지 (문제 기준: 북, 북서, 서, 남서, 남, 남동, 동, 북동)
    static int[] dr = {-1, -1, 0, 1, 1, 1, 0, -1};
    static int[] dc = {0, -1, -1, -1, 0, 1, 1, 1};

    static class Fish {
        int r, c, dir;
        boolean isAlive;

        Fish(int r, int c, int dir, boolean isAlive) {
            this.r = r; this.c = c; this.dir = dir; this.isAlive = isAlive;
        }
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int[][] board = new int[4][4];
        Fish[] fishes = new Fish[17];

        for (int i = 0; i < 4; i++) {
            for (int j = 0; j < 4; j++) {
                int num = sc.nextInt();
                int dir = sc.nextInt() - 1; // 0-index로 변경
                board[i][j] = num;
                fishes[num] = new Fish(i, j, dir, true);
            }
        }

        // 초기 상어 진입: (0,0) 물고기를 먹음
        int firstFishNum = board[0][0];
        Fish firstFish = fishes[firstFishNum];
        int sharkR = 0, sharkC = 0, sharkDir = firstFish.dir;
        firstFish.isAlive = false;
        board[0][0] = -1; // 상어 위치 표시

        solve(board, fishes, sharkR, sharkC, sharkDir, firstFishNum);
        System.out.println(maxScore);
    }

    static void solve(int[][] board, Fish[] fishes, int sR, int sC, int sD, int sum) {
        maxScore = Math.max(maxScore, sum);

        // 1. 데이터 Deep Copy (원상복구 대신 복사본 사용)
        int[][] cBoard = new int[4][4];
        for (int i = 0; i < 4; i++) cBoard[i] = board[i].clone();
        Fish[] cFishes = new Fish[17];
        for (int i = 1; i <= 16; i++) {
            cFishes[i] = new Fish(fishes[i].r, fishes[i].c, fishes[i].dir, fishes[i].isAlive);
        }

        // 2. 물고기 이동
        moveFish(cBoard, cFishes);

        // 3. 상어 이동 (최대 3칸 점프 가능)
        for (int dist = 1; dist < 4; dist++) {
            int nr = sR + dr[sD] * dist;
            int nc = sC + dc[sD] * dist;

            if (nr >= 0 && nr < 4 && nc >= 0 && nc < 4 && cBoard[nr][nc] > 0) {
                int fishNum = cBoard[nr][nc];
                int nextDir = cFishes[fishNum].dir;

                // 상태 변경
                cBoard[sR][sC] = 0; // 기존 상어 자리 빈칸
                cBoard[nr][nc] = -1; // 상어 이동
                cFishes[fishNum].isAlive = false;

                solve(cBoard, cFishes, nr, nc, nextDir, sum + fishNum);

                // 복구 (다음 dist를 위해 이 단계의 cBoard/cFishes 상태를 되돌림)
                cFishes[fishNum].isAlive = true;
                cBoard[nr][nc] = fishNum;
                cBoard[sR][sC] = -1;
            }
        }
    }

    static void moveFish(int[][] board, Fish[] fishes) {
        for (int i = 1; i <= 16; i++) {
            if (!fishes[i].isAlive) continue;

            Fish cur = fishes[i];
            for (int d = 0; d < 8; d++) {
                int nDir = (cur.dir + d) % 8;
                int nr = cur.r + dr[nDir];
                int nc = cur.c + dc[nDir];

                // 경계 안이고 상어가 아니면 이동 가능
                if (nr >= 0 && nr < 4 && nc >= 0 && nc < 4 && board[nr][nc] != -1) {
                    board[cur.r][cur.c] = 0;
                    if (board[nr][nc] == 0) { // 빈칸 이동
                        cur.r = nr; cur.c = nc;
                    } else { // 다른 물고기와 교체
                        int targetNum = board[nr][nc];
                        fishes[targetNum].r = cur.r;
                        fishes[targetNum].c = cur.c;
                        cur.r = nr; cur.c = nc;
                        board[fishes[targetNum].r][fishes[targetNum].c] = targetNum;
                    }
                    cur.dir = nDir;
                    board[nr][nc] = i;
                    break;
                }
            }
        }
    }
```

```java
import java.util.*;

class Solution {
    /**
    # 해석 & 접근
    attacks 에 따른 bandage 와 health 로직 처리하여 최종 health 반환, 다만 죽었다면 -1 처리
	-> attacks 에 대한 map 구조로 만든 뒤, 0부터 attacks 최대 key 까지 순회하며 attack 처리
    -> map 활용한 단순 구현 문제
    공격 당한 후 붕대감기 연속성공 처리
    -> 공격 시점이 아니라면 연속성공 카운트 ++
    -> 연속성공 카운트가 시전 시간과 일치하면 추가힐링
    -> 공격 시점이라면 연속성공 카운트 := 0

    # 복잡도
    시간 : attack 순회 -> O(N)
    공간 : attack 을 map 으로 바꾸는 일 -> O(attack 의 크기)
    **/
    public int solution(int[] bandage, int health, int[][] attacks) {
        Map<Integer,Integer> attackMap = new HashMap<>();
        for (int[] attack : attacks) {
            attackMap.put(attack[0],attack[1]);
        }

        int iterationCount = Collections.max(attackMap.keySet());
        int bandThreshold = bandage[0];
        int healPerSec = bandage[1];
        int healExtra = bandage[2];
        int healCount = 0;
        int maxHealth = health;

       	for (int i=1; i<=iterationCount; i++){
            int attack = attackMap.getOrDefault(i,0);
            boolean isAttack = false;

            if(attack != 0){
                isAttack = true;
            }

            // 공격당함 -> 체력회복 카운트 := 0, 체력 삭감, 다음 iteration 시작
            if(isAttack) {
                health -= attack;
            	healCount = 0;
                if(health <= 0) {
                    return -1;
                }
                continue;
            }

            // 초당 체력회복
            // 연속 체력회복 O -> 추가체력회복 이후 체력회복 카운트 초기화
            // 최대 체력 유지
            health += healPerSec;
            healCount++;
            if(healCount == bandThreshold){
                health += healExtra;
                healCount = 0;
            }
            health = Math.min(health, maxHealth);
        }

        return health;
    }
```

```java
import java.util.*;

class Solution {
    /**
    분, 초 처리 -> 분, 초를 split 해서 변수로 따로 , 초가 +60 이 넘어가면 while 과 /60 을 통해 posMin ++, -60 넘어가면 똑같이
    next, prev 처리 -> next 라면 +10, prev 라면 -10
    오프닝 점프 -> op_start <= pos <= op_end 이라면 pos := op_end
    **/
        public String solution(String video_len, String pos, String op_start, String op_end, String[] commands) {
        int videoLenSec = toSecond(video_len);
        int posSec = toSecond(pos);
        int opStartSec = toSecond(op_start);
        int opEndSec = toSecond(op_end);

        posSec = skipOpening(posSec, opStartSec, opEndSec);

        for (String command : commands) {
            if (command.equals("next")) {
                posSec += 10;
                if (posSec > videoLenSec) posSec = videoLenSec;
            } else {
                posSec -= 10;
                if (posSec < 0) posSec = 0;
            }

            posSec = skipOpening(posSec, opStartSec, opEndSec);
        }

        return String.format("%02d:%02d", posSec / 60, posSec % 60);
    }

    private int toSecond(String time) {
        String[] temp = time.split(":");
        return Integer.parseInt(temp[0]) * 60 + Integer.parseInt(temp[1]);
    }

    private int skipOpening(int current, int start, int end) {
        if (current >= start && current <= end) {
            return end;
        }
        return current;
    }
}
```

# DP/Memoization

---

### 테크닉 : Top Down / Bottom Up

하나의 문제를 작은 문제로 나누어 해결하는데 이 때 base case 처리시점에 따라 다르게 처리할 수 있다.

base case 부터 차근차근 for/while 을 통해 해결해나가는 게 Bottom Up,

큰 수부터 수를 줄여가며 base case 까지 재귀호출해나가는 게 Top Down 방식이다.

여기서 Memoization 은 중간에 계산한 값을 블룸 필터와 같이 map 형태나 arr[index] 형태로 저장하여

다시 연산하지 않게 하는 방식이다.

다만 RISC,CISC 구조와 언어에 따라 Memoization 이 항상 빠른 게 아닐 수 있으므로 유의해야한다

만약 비트연산이 가능하다면 Memoization 보다 비트연산이 더 빠를 수 있기 때문이다.

### 테크닉 : Memo vs Greedy

Greedy 와 차이점은 문제 갯수를 줄이느냐 여부이다

Greedy 는 최적해를 찾기 위해 현재 상태에서 가장 좋아 보이는 것을 선택하고 뒤돌아보지 않는다

DP 는 현재의 선택이 나중에도 영향을 미치기 때문에, 이전의 모든 결과값을 고려하여 최적해를 쌓아나간다

### 테크닉 : 2-fingers DP (상태: idx, left, right)

> - 선택지가 “왼손 vs 오른손”으로 계속 갈라지는 문제는 상태를 고정해 메모이제이션
> - `memo[idx][left][right]` : `idx`부터 끝까지의 최소 비용

### 문제 : [Minimum Distance to Type a Word Using Two Fingers](https://leetcode.com/problems/minimum-distance-to-type-a-word-using-two-fingers/)

> 요구사항
>
> - 두 손가락으로 키보드(6열)에서 `word`를 입력할 때 최소 이동 거리

> 접근/원리
>
> - 상태: `(idx, leftPos, rightPos)` / 다음 글자를 왼손으로 치는 경우 vs 오른손으로 치는 경우의 최소
> - 미사용 손가락은 sentinel(예: 26)로 두고 이동비용 0 처리

```java
import java.util.*;

class Solution {
    private int[][][] memo;

    public int minimumDistance(String word) {
        int n = word.length();
        memo = new int[n][27][27];
        for (int i = 0; i < n; i++) {
            for (int a = 0; a < 27; a++) {
                Arrays.fill(memo[i][a], -1);
            }
        }
        return dfs(0, 26, 26, word);
    }

    private int dfs(int idx, int left, int right, String word) {
        if (idx == word.length()) return 0;
        if (memo[idx][left][right] != -1) return memo[idx][left][right];

        int next = word.charAt(idx) - 'A';

        int costLeft = (left == 26 ? 0 : dist(left, next)) + dfs(idx + 1, next, right, word);
        int costRight = (right == 26 ? 0 : dist(right, next)) + dfs(idx + 1, left, next, word);

        return memo[idx][left][right] = Math.min(costLeft, costRight);
    }

    private int dist(int a, int b) {
        int x1 = a / 6, y1 = a % 6;
        int x2 = b / 6, y2 = b % 6;
        return Math.abs(x1 - x2) + Math.abs(y1 - y2);
    }
}
```

### 문제 : [Minimum Total Distance Traveled](https://leetcode.com/problems/minimum-total-distance-traveled/)

> 요구사항
>
> - 로봇 위치 `robot[]`을 공장 `factory[i] = [pos, limit]`에 배정해 이동 거리 합 최소화

> 접근/원리
>
> - 로봇/공장을 좌표순 정렬
> - 각 공장에 0..limit개의 로봇을 배정하는 선택지를 DP로 누적

```java
import java.util.*;

class Solution {
    public long minimumTotalDistance(int[] robot, int[][] factory) {
        Arrays.sort(robot);
        Arrays.sort(factory, (a, b) -> Integer.compare(a[0], b[0]));

        long INF = 1_000_000_000_000L;
        int m = robot.length;
        int f = factory.length;

        long[][] dp = new long[f + 1][m + 1];
        for (int i = 0; i <= f; i++) {
            Arrays.fill(dp[i], INF);
            dp[i][0] = 0;
        }

        for (int i = 0; i < f; i++) {
            int pos = factory[i][0];
            int limit = factory[i][1];

            for (int j = 0; j <= m; j++) {
                // 현재 공장을 쓰지 않는 선택
                dp[i + 1][j] = Math.min(dp[i + 1][j], dp[i][j]);

                long cost = 0;
                for (int k = 1; k <= limit && j + k <= m; k++) {
                    cost += Math.abs((long) robot[j + k - 1] - pos);
                    if (dp[i][j] != INF) {
                        dp[i + 1][j + k] = Math.min(dp[i + 1][j + k], dp[i][j] + cost);
                    }
                }
            }
        }

        return dp[f][m];
    }
}
```

# Dijkstra

---

BFS / DFS 는 각 노드들을 탐색해나가는 방법이다.

따라서 BFS / DFS 로 특정노드까지 탐색이 가능하다

다만 가중치가 1 이라는 전제이다

Dijkstra 는 가중치가 다른 그래프의 노드들에서 최소 거리 탐색을 한다.

다만 가중치가 **음수**일 때는 사용 불가하다(이때는 벨만-포드를 쓰지만, 코테에선 거의 안 나온다)

- 시작 노드의 거리를 0으로 두고 PQ에 넣는다.
- PQ에서 거리가 가장 짧은 노드를 꺼낸다.
- 그 노드를 거쳐서 인접 노드로 가는 게 기존 거리보다 짧으면? `dist[]` 갱신하고 PQ에 넣는다.

```java
import java.util.*;

class Node implements Comparable<Node> {
    int index;
    int distance;

    Node(int index, int distance) {
        this.index = index;
        this.distance = distance;
    }

    // 우선순위 큐에서 거리가 짧은 순으로 정렬하기 위함
    @Override
    public int compareTo(Node other) {
        return Integer.compare(this.distance, other.distance);
    }
}

public class DijkstraExample {
    static final int INF = Integer.MAX_VALUE;
    static List<List<Node>> graph = new ArrayList<>();
    static int[] dist;

    public void dijkstra(int n, int start) {
        dist = new int[n + 1];
        Arrays.fill(dist, INF);

        // 1. 시작 노드 초기화
        PriorityQueue<Node> pq = new PriorityQueue<>();
        dist[start] = 0;
        pq.offer(new Node(start, 0));

        while (!pq.isEmpty()) {
            // 2. 현재 가장 거리가 짧은 노드 꺼내기
            Node curr = pq.poll();
            int d = curr.distance;
            int now = curr.index;

            // 이미 방문했거나 현재 거리가 더 짧은 경로가 있다면 무시 (중요!)
            if (dist[now] < d) continue;

            // 3. 인접 노드 확인
            for (Node next : graph.get(now)) {
                int cost = dist[now] + next.distance;

                // 4. 거리가 더 짧으면 갱신하고 PQ에 삽입
                if (cost < dist[next.index]) {
                    dist[next.index] = cost;
                    pq.offer(new Node(next.index, cost));
                }
            }
        }
    }
}
```

## 테크닉 : Binary Search

> 💡 핵심 원리
>
> - **정렬된 배열**에서 타겟 데이터를 찾기 위해 탐색 범위를 **절반씩 좁혀나감**
>     - 왼쪽,중앙,오른쪽에 대한 포인터 변수를 선언(`low`,`mid`,`high`), 타겟값을 비교하여
>       중앙값이 타겟보다 크면 오른쪽 구간 선택 -- `low = mid + 1`
>       중앙값이 타겟이 작으면 왼쪽 구간 선택 -- `high = mid - 1`
>     - 이 때 `mid` 연산 시 $mid = (low + high) / 2$ 로 연산하게 되면
>       두 값을 더하는 순간 `int` 범위를 넘어서 오버플로우 발생할 수 있음
>       이를 방지하기 위해 `low + 중간까지거리` 인
>       $mid = low + (high - low) / 2$ 로 연산함
> - 주의해야할 점은 정렬된 배열에서 사용해야한다는 점
>   따라서 사전작업인 정렬을 위해 O($NlogN$) 이 추가로 들어간다
> - 탐색 범위가 매 단계마다 1/2 씩 줄어든다
>   따라서 시간 복잡도는 O($\log_2 n$)  
>   따라서 특정 값을 탐색해야할 때 선형탐색 대신 사용하면 복잡도 개선이 가능하다

```java
int binarySearch(int[] arr, int target) {
    int low = 0;
    int high = arr.length - 1;

    while (low <= high) {
        int mid = low + (high - low) / 2; // 오버플로우 방지

        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            low = mid + 1;
        } else {
            high = mid - 1;
        }
    }
    return -1;
}
```



## 문제 : [**Search in Rotated Sorted Array**](https://leetcode.com/problems/search-in-rotated-sorted-array/)

> 요구사항
>
> - (언어, 직군, 경력, 소울푸드)과 점수를 가진 유저 데이터 info 배열이 주어짐
> - 특정 조건과 최소 점수를 만족하는 지원자 수를 묻는 query 배열
> - 각 쿼리에 만족하는 지원자 수를 계산하여 반환
> - info 의 길이 최대는 50,000 , query 의 길이 최대는 100,000

> 접근/원리
>
> - 단순 Stream API 사용한 완전탐색은 비효율 접근
>     - 쿼리마다 구조체를 탐색하면 N x M 으로 처리되므로 50k x 100k
>     - 따라서 시간초과가 발생할 수 있음
> - 해결안에 대해서는 두 가지 테크닉이 들어감
>     - 각 유저가 해당하는 조건을 조합하여 Map 에 저장, query 에 대해 O(1) 처리하도록 제공 :: O(N)
>     - 쿼리가 요구하는 `x 점 이상의 사람수` 를 구하기 위해 query 에 질의하는 점수를 Lower Bound 를 두고 정렬 & 이진탐색 처리 :: 정렬 O(NlogN) + 쿼리별이진탐색 O(N) \* O(logN)
>     - 따라서 최종 시간복잡도는 O(N + NlogN + NlogN) 으로 처리됨

```java
import java.util.*;

class Solution {
    // 쿼리 조건 문자열을 Key로, 해당 조건 유저들의 점수 리스트를 Value로 관리
    Map<String, List<Integer>> map = new HashMap<>();

    public int[] solution(String[] info, String[] query) {
        // 1. 모든 info를 순회하며 유저당 16가지 조합을 만들어 Map에 점수 저장
        for (String i : info) {
            String[] split = i.split(" ");
            int score = Integer.parseInt(split[4]);
            makeCombinations(split, "", 0, score);
        }

        // 2. 이진 탐색을 위해 Map의 모든 점수 리스트를 오름차순 정렬
        for (List<Integer> list : map.values()) {
            Collections.sort(list);
        }

        int[] answer = new int[query.length];

        // 3. 쿼리를 순회하며 Map 탐색 + 이진 탐색 조합으로 정답 도출
        // " and " 문자열 파싱 에러 방지를 위해 미리 공백 처리
        // 이진 탐색으로 targetScore 이상인 점수가 시작되는 인덱스를 찾음
        for (int i = 0; i < query.length; i++) {
            String cleanQuery = query[i].replace(" and ", "");
            String[] split = cleanQuery.split(" ");

            String queryKey = split[0];
            int targetScore = Integer.parseInt(split[1]);

            if (map.containsKey(queryKey)) {
                List<Integer> scoreList = map.get(queryKey);
                int index = binarySearch(scoreList, targetScore);
                answer[i] = scoreList.size() - index;
            } else {
                answer[i] = 0;
            }
        }

        return answer;
    }

    // DFS 백트래킹을 활용해 한 유저가 매칭될 수 있는 모든(-) 조건 키 생성
    private void makeCombinations(String[] split, String current, int depth, int score) {
        if (depth == 4) {
            map.computeIfAbsent(current, k -> new ArrayList<>()).add(score);
            return;
        }
        // Case A: 원래 조건 문자열 이어 붙이기
        makeCombinations(split, current + split[depth], depth + 1, score);
        // Case B: 조건 없음("-") 문자열 이어 붙이기
        makeCombinations(split, current + "-", depth + 1, score);
    }

    // Lower Bound 이진 탐색: target 이상의 값이 처음 나오는 인덱스 탐색
    private int binarySearch(List<Integer> list, int target) {
        int low = 0;
        int high = list.size() - 1;

        while (low <= high) {
            int mid = low + (high - low) / 2;

            if (list.get(mid) < target) {
                // 타겟보다 작으면 무조건 우측 구간으로
                low = mid + 1;
            } else {
                // 타겟 이상이면 범위를 좁히기 위해 좌측 구간으로
                high = mid - 1;
            }
        }
        return low;
    }
}
```

## 문제 : [**Search in Rotated Sorted Array**](https://leetcode.com/problems/search-in-rotated-sorted-array/)

> 요구사항
>
> - 0 ~ K 까지 정렬, K+1 ~ N 까지 정렬된 nums 배열이 있다 — e.g. [4,5,6,7,0,1,2]
> - 이에 대해서 주어진 target 이 nums 의 어느 인덱스에 있는지 찾아라
> - 주어진 제한조건은 O(logN) 으로 풀어야하며 nums 의 길이가 5000, 각각의 숫자의 최대는 10^4

> 접근/원리
>
> - logN 으로 풀어야하므로 정렬,완탐은 사용 불가
> - half, half 지점을 찾아서 이진탐색을 수행해야함
> - 이 때 half, half 지점을 순회해서 찾으려면 N 시간이 걸림
>   따라서 target 찾는 이진탐색 동시에 함

```java
class Solution {
    public int search(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;

        while(left <= right){
            int mid = left + (right - left) / 2;
            if(nums[mid] == target){
                return mid;
            }

            // 왼쪽 half 정렬되어있다면
            // target 이 왼쪽 안에 있는지 여부에 따라
            // right, left 이동
            if(nums[left] <= nums[mid]){
                if(nums[left] <= target && target < nums[mid]){
                    right = mid - 1;
                }
                else{
                    left = mid + 1;
                }
            }
            // 오른쪽 half 정렬되어있다면
            // target 이 오른쪽 안에 있는지 여부에 따라
            // right, left 이동
            else{
                if(nums[mid] < target && target <= nums[right]){
                    left = mid + 1;
                }
                else{
                    right = mid - 1;
                }
            }
        }

        return -1;
    }
}
```

## 문제 : [Maximize the Distance Between Points on a Square](https://leetcode.com/problems/maximize-the-distance-between-points-on-a-square/)

> 요구사항
>
> - 변의 길이가 `side`인 정사각형 테두리 위에 여러 개의 점이 주어집니다.
> - 이 중 **`k`개의 점을 선택**했을 때, 선택된 점들 사이의 **최소 맨해튼 거리(Min Distance)를 최대화(Maximize)** 하는 값을 찾아야 합니다.

> 접근/원리
>
> - 선상을 그룹핑하는 접근이 아닌 이유 (UnionFind / Set ,,,)
>     - 그룹핑을 하려면 "어떤 거리 이내면 같은 그룹인가?"라는 **고정된 기준**이 필요함
>     - 하지만 기준거리를 구하려면 모든 노드를 탐색해야하고 이로 인한 시간초과 발생함
> - 투포인터가 아닌 이유
>     - 투포인터는 보통 `left`와 `right`라는 두 개의 지점을 조절하며 탐색해야함
>     - 하지만 이 문제는 점을 **k개 골라야하므로 적합하지 않음**
> - Binary Search 를 떠올릴 수 있는 Decsion Tree
>     - `1`부터 `2 * side` 사이에서 d 를 순회하며
>       k 개를 놓을 수 있는지 Greedy 하게 찾으면 되겠구나!
>       → 이는 탐색범위가 정해졌고
>       → 단조적으로 Greedy 판단이 가능하기에 접근가능함!
>       → 즉 이 문제는 **Maximize the Minimum 유형!**
>     - 최적조건에 대한 구현은 다음과 같이 처리됨
>         - 직사각형에 대한 2차원배열을 1차원으로 평탄화 — (0,0)에서 출발하여 시계 방향으로 테두리를 따라가며 거리를 계산
>             - 밑변(y=0) → x
>             - 우변(x=side) → side + y
>             - 윗변(y=side) → 2L \* side + (side - x)
>             - 좌변(x=0) → 3L \* side + (side - y)
>         - 이후 greedy 를 통해 최적조건 판단
>             - 0번 점부터 거리 d 이내에 있는 점들만 시작 후보
>             - 현재 점에서 최소 d 만큼 떨어진 가장 가까운 다음 점을 선택하며 k개 채움
>             - 마지막 선택 점과 시작점 사이의 거리(전체 길이 - 직선거리)가 d 이상인지 확인

```java
import java.util.*;

class Solution {
    public int maximizeSquareDistance(int side, int[][] points, int k) {
        int n = points.length;
        long[] linearPositions = new long[n];

        // 1. 2D 좌표를 테두리 길이 기반 1차원 좌표로 변환
        for (int i = 0; i < n; i++) {
            linearPositions[i] = convertToLinear(points[i], side);
        }
        Arrays.sort(linearPositions);

        // 2. 이진 탐색 (최소 거리 d를 최대로 만들기)
        int low = 1, high = 2 * side;
        while (low <= high) {
            int midDistance = low + (high - low) / 2;
            if (canPlaceAllPoints(linearPositions, k, midDistance, 4L * side)) {
                low = midDistance + 1; // 더 큰 거리 시도
            } else {
                high = midDistance - 1;
            }
        }
        return high;
    }

    private long convertToLinear(int[] point, int side) {
        int x = point[0], y = point[1];
        if (y == 0) return x;                         // 밑변
        if (x == side) return (long) side + y;        // 우변
        if (y == side) return 2L * side + (side - x); // 윗변
        return 3L * side + (side - y);                // 좌변
    }

    private boolean canPlaceAllPoints(long[] positions, int k, int minDistance, long totalBoundary) {
        int n = positions.length;

        // 원형 구조 최적화: 시작점은 첫 점으로부터 minDistance 이내인 것들만 확인
        for (int start = 0; start < n && (positions[start] - positions[0] < minDistance); start++) {
            int currentIdx = start;
            int count = 1;

            // k-1개의 다음 점들을 이진 탐색으로 빠르게 찾기
            for (int j = 1; j < k; j++) {
                currentIdx = findNextPointIdx(positions, positions[currentIdx] + minDistance, currentIdx + 1);

                // 더 이상 점을 찍을 수 없거나, 한 바퀴를 넘어서 시작점을 침범하면 중단
                if (currentIdx == n || positions[currentIdx] + minDistance - totalBoundary > positions[start]) {
                    count = -1;
                    break;
                }
            }

            // k개를 다 찍었고, 마지막 점과 시작점 사이의 거리(Wrap-around)도 보장되면 성공
            if (count == 1 && (positions[currentIdx] + minDistance - totalBoundary <= positions[start])) {
                return true;
            }
        }
        return false;
    }

    // 현재 위치 + minDistance 이상인 가장 가까운 점의 인덱스를 이진 탐색으로 찾음 (Lower Bound)
    private int findNextPointIdx(long[] positions, long targetValue, int fromIdx) {
        int low = fromIdx, high = positions.length;
        while (low < high) {
            int mid = low + (high - low) / 2;
            if (positions[mid] < targetValue) low = mid + 1;
            else high = mid;
        }
        return low;
    }
}
```

## 테크닉 : Parametric Search
> 💭 **Parametric Search 를 떠올리는 Thought Process**
> 1. Maximize the Minimum 형태의 질문이라면 바로 Parametric Search 을 떠올릴 수 있다
> 	1. `~~할 수 있는 최솟값의 최댓값을 구하시오`
> 	2. `~을 만족하는 최소 비용 / 최소 시간을 구하시오`
> 2. 주어진 N 의 크기를 보고 힌트를 얻는다
> 	1. 완전탐색은 가능한가?
> 	2. 그리디하게 처리가 가능한가?
> 	3. 점화식으로 처리가 가능한가?
> 	4. **값을 찾고 조건에 만족하는지 찾아가는 Parametric Search 가 가능한가?** -- 이 때 좌우로 Max/Min Bound 를 찾을 수 있는 단조성(Monotone) 이라면 OK
> 주어진 N 에 따라 Parametric Search 가 가능하다면 시간복잡도를 연산하고
> 가능한 것 같으면 구현을 해보고 테스트를 돌려보자

> 💡 **핵심 원리**
> - **연속된 정답의 후보 범위**에서 최적의 답을 찾기 위해 탐색 범위를 **절반씩 좁혀나감**
    - 중앙값(`mid`)을 "이게 정답일까?"라고 가정하고, 결정 함수 `isValid(mid)`를 실행함
    - 만약 **최솟값의 최대화** 문제에서 `mid`가 유효하다면:  
        더 큰 값도 가능한지 보기 위해 오른쪽 선택 -- `low = mid + 1` (동시에 현재 `mid`를 `answer`에 백업)  
        만약 `mid`가 유효하지 않다면:  
        기준을 낮춰야 하므로 왼쪽 선택 -- `high = mid - 1`
    - `mid` 연산 시 오버플로우 방지를 위해 \(mid = low + (high - low) / 2\) 를 사용하는 것은 동일함
> - 주의해야할 점은 **결정 함수의 결과가 단조성(Monotonicity)** 을 띄어야 한다는 점 (`[T, T, T, F, F]` 구조)
> - 탐색 범위가 매 단계마다 1/2 씩 줄어들므로 시간 복잡도는 O($\log_2 (high-low) * isValid()$) 가 된다.

```java
int parametricSearch(int minCandidate, int maxCandidate) {
    int low = minCandidate;
    int high = maxCandidate;
    int answer = -1; // 최적의 정답을 저장할 변수

    while (low <= high) {
        int mid = low + (high - low) / 2; // 오버플로우 방지

        // mid라는 값으로 문제의 조건을 만족할 수 있는가? (결정 함수)
        if (isValid(mid)) { 
            answer = mid;     // 조건을 만족하므로 일단 정답 후보로 저장
            low = mid + 1;    // 최솟값의 최대화이므로, 더 큰 값도 가능한지 보러 우측 탐색
        } else {
            high = mid - 1;   // 불가능하므로 기준을 낮추러 좌측 탐색
        }
    }
    return answer;
}
```

## 문제 : [징검다리 (Parametric Search)](https://school.programmers.co.kr/learn/courses/30/lessons/43236)

> 요구사항
> - 출발선부터 도착선까지의 거리 `distance`, 그 사이에 있는 바위들의 위치 배열 `rocks`, 제거할 바위의 개수 `n`ㅎ> - 바위를 정확히 n개 제거했을 때, 바위 간 최소 거리(Min Distance)의 최댓값(Maximize)**을 구해야 함
> - 출발선부터 도착선까지의 거리는 최대 $10^9$(10억), 바위의 개수는 최대 50,000

> 접근/원리
> - **단순 그리디(가장 짧은 구간 순차 제거)가 실패하는 이유**
>     - 바위를 하나 지우면 양옆의 두 거리가 합쳐져 새로운 하나의 거리가 됨. 
> 	   즉, 이전 단계의 선택이 다음 단계의 거리 구조를 완전히 변형시킴.
>     - 근시안적으로 당장 짧은 거리만 지워나가는 방식은 
> 	   최종적인 '합쳐진 거리들의 시너지'를 고려하지 못하므로 오답을 유발함.
> - **Binary Search를 떠올릴 수 있는 Decision Tree**
>     - 바위를 조합으로 고르는 완전탐색($2^{50,000}$)은 당연히 불가능.
>     - 질문을 역발상으로 뒤집어 
> 	   **"최소 거리가 $M$ 이상이 되도록 만들려면 바위를 몇 개 지워야 하는가?"** 
> 	   라는 가설 검증(Decision) 문제로 전환.
> 	- 즉, 이 문제는 **Parametric Search(Maximize the Minimum)** 유형
> 		- 출발선부터 시작해서 다음 바위까지의 거리를 잼
> 		- 만약 그 거리가 `mid`보다 **작다면**, 가설(`mid` 이상이어야 함)을 만족하지 못하므로 **그 바위를 제거**. (제거 카운트 + 1)
> 		- 만약 거리가 `mid`보다 **크거나 같다면**, 가설을 만족하므로 바위를 살려두고 기준으로 삼음
> 		- 이 작업을 끝까지 마쳤을 때 **제거한 바위의 개수가 $n$개 이하**라면? 
> 			-> `mid`라는 최소 거리를 만드는 것이 가능하다는 뜻!
> - **최종 시간 복잡도**
>     - 정렬 비용 $O(R \log R)$ + 이진 탐색 비용 $O(R \log(\text{Distance}))$ :: 여기서 $R = \text{rocks.length}$
>     - $\log_2(10^9)$은 치트키($10^3 \approx 2^{10}$)에 의해 약 **30번**의 루프로 환산됨.
>     - 최종 연산 횟수는 정렬($50,000 \times 16$) + 탐색($30 \times 50,000$) $\approx 2.3 \times 10^6$번으로, 제한 시간 내에 완벽하게 통과 가능.

```java
import java.util.*;

class Solution {
    /**
    n 개의 노드 제거에 따른 노드 간 거리 최소값 중 최대값 반환
    완점탐색으로 풀 수 없음 -- visited 를 통해 바위 제거 조합 n^n(25 * 10^8) * 각 바위 사이 거리 재기 n
    노드 간 관계도 없어서 그래프로 접근은 안 됨
    각 dist 를 정렬하여 그리디로 접근하고자해도
    바위 삭제에 따른 거리가 재초기화되며 합쳐진 거리를 재연산해야하는
    이슈를 해결해야하므로 불가능
    
    최소 거리가 $M$ 이상이 되도록 만들려면 바위를 몇 개 지워야 하는지로
    즉 Parametric Search 로 접근
    바위 사이 거리를 탐색 축으로 mid 라는 기준거리를 두고
    기준거리보다 짧은 구간이 나타나면 탐색에서 제외하며 제거 카운트를 누적
    **/
    public int solution(int distance, int[] rocks, int n) {
        Arrays.sort(rocks);
        
        int left = 1;
        int right = distance;
        int answer = 0;
        
        while(left <= right){
            int midDist = left + (right - left) / 2;
            int removed = 0;
            int curr = 0;
            
            // 모든 바위 순회하며 기준보다 짧은 바위들을 제거
            // 마지막 바위와 도착점 사이 최종 거리도 검증
            for(int i=0;i<rocks.length;i++){
                int currDist = rocks[i] - curr;
                if(currDist < midDist){
                    removed++;
                    continue;
                }
                curr = rocks[i];
            }
            if(distance - curr < midDist){
                removed++;
            }
            
            // LowerBound 를 활용한 탐색 범위 조작
            // n 개 이하로 지워서 거리 보장 가능하다면 유효 -- 정답 후보로 기록 후 최소거리 극대화를 위한 우측구간 선택하여 탐색
            // n 개 초과로 지워야한다면 불가능 -- 정답 찾기 위해 좌측 구간 탐색
            if(removed <= n){
                answer = midDist;
                left = midDist + 1;
                continue;
            }
            right = midDist - 1;
        }
        
        return answer;
    }
}
```

# Two Pointer

---

2개의 포인터(인덱스 변수) 를 사용해서 배열을 훑는 기법이다
보통 정렬된 배열에서 특정 합을 찾거나, 연속된 구간의 특징을 찾을 때 사용한다
double for loop 를 사용하면 O(N^2) 으로 인해 시간초과가 날 때 사용한다
투포인터는 N 번만 훑으므로 O(N) 으로 끝나기 때문이다.

## 문제 : 두수의 합

https://www.acmicpc.net/problem/3273

> ⭐
>
> 왜 투포인터로 풀어야하는가?
> → 완전탐색의 접근을 떠올릴 수 있으나 n 이 100000 이라서 시간초과가 난다.
> → 연속된 수가 아닌 서로 다른 수의 합산이므로 슬라이딩 윈도우는 사용할 수 없다.
> → 범위로 접근하지 말고, 특정 숫자를 지정하는 변수를 통해 경우의 수를 찾는다, 즉 투포인터를 활용한다.

> ⭐
>
> 조건 `ai + aj = x (1 ≤ i < j ≤ n)을 만족` 에 대해서는 어떻게 처리하는가?
> → 이 조건은 수의 순서가 중요한 게 아니라 두 개의 수가 서로 다른 수이며 중복되지 않아야한다는 조건이다
> → .e.g. arr[3] + arr[3] ❌, arr[3] + arr[1] ❌
> → .e.g. arr[1] + arr[3] ✅

> ⭐
>
> 해당 문제에서 투 포인터 접근 전략이 어떻게 되는가?
> → 정렬 이후 양 쪽에 포인터를 잡고, 두 수의 합을 비교한다.
> → 만약 합이 목표값보다 작다면 더 큰 값을 더해주기 위해 시작 포인터 이동
> → 만약 합이 목표값보다 크다면 더 작은 값을 더해주기 위해 끝 포인터 이동
> → 만약 합이 목표값보다 같다면 count 해주고, 두 포인터를 안쪽으로 옮겨서 새로운 조합을 찾는다

```java

  /**
   * n 이 100000 이라서 재귀호출을 통해 완전탐색은 어려울 듯 하다
   *
   * <p>따라서 투 포인터로 접근한다
   *
   * <p>정렬한 뒤 start, end 를 양끝으로 잡고
   *
   * <p>start, end 를 가운데로 몰아가며 count 계산
   */
  public static int resolve(int n, int[] arr, int x) {
    Arrays.sort(arr);

    int startPtr = 0;
    int endPtr = arr.length - 1;

    int count = 0;
    while (startPtr < endPtr) {
      int sum = arr[startPtr] + arr[endPtr];
      if (sum == x) {

        count++;
        startPtr++;
        endPtr--;
        continue;
      }
      if (sum < x) {
        startPtr++;
        continue;
      }
      endPtr--;
    }

    return count;
  }
```
