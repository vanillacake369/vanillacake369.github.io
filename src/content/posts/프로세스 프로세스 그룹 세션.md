# **프로세스**

- 실행 중인 프로그램의 인스턴스이며, 시스템 리소스 할당의 기본 단위이다.
- 고유한 주소 공간과 하나 이상의 스레드를 가진다.
- 부모–자식 계층을 이루며, 자식은 부모의 여러 속성을 상속한다.

# **프로세스 그룹**

- 관련 프로세스들의 묶음이다. PGID는 그룹 리더의 PID와 같다.
- `setpgid()/setpgrp()`로 생성·변경한다.
- 리더는 다른 그룹에 가입할 수 없고, 구성원이 하나라도 있으면 그룹은 유지된다.
- 부모는 PGID 단위로 `wait()` 할 수 있고, 신호는 `kill(-PGID, SIG)` 또는 `killpg()`로 그룹 전체에 보낸다.

# **세션**

- 프로세스 그룹들의 묶음이다. SID는 세션 리더의 PID와 같다.
- `setsid()`로 새 세션을 시작하며 동시에 새 프로세스 그룹의 리더가 된다.
- 세션 구성원은 하나의 제어 터미널을 공유하며, Ctrl-C 등 터미널 신호는 세션의 포어그라운드 프로세스 그룹으로 전달된다.
- 로그인 셸과 데몬이 대표적으로 독립 세션에서 실행된다.
