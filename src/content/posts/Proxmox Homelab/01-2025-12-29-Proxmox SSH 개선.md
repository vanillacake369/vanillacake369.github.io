# Why? 🤔

Proxmox는 설치 직후부터 기본적으로 root 계정의 SSH 접속을 허용하고 있다. 하지만 기본 설정 그대로 두는 것보다 보안을 위해 다음 설정들을 처리해야 한다.[^1]

- 로그인 방식 변경
- 관리 전용 사용자
- Fail2Ban

기존 SSH 방식이 왜 위험한지, 위 설정들은 무엇인지, 어떻게 하면 설정할 수 있는지를 알아본다.

### 설정 시 발생한 문제 & 해결법

기본 SSH 연결 접속 시도 시 22번이 아닌 22327 로 접근하려는 문제가 발생했었다.

```bash
❯  ssh -v -i proxmox.pem root@192.168.45.82 2>&1
Warning: Identity file proxmox.pem not accessible: No such file or directory.
OpenSSH_9.9p2, LibreSSL 3.3.6
debug1: Reading configuration data /Users/limjihoon/.ssh/config
debug1: Reading configuration data /etc/ssh/ssh_config
debug1: Reading configuration data /etc/ssh/ssh_config.d/100-macos.conf
debug1: /etc/ssh/ssh_config.d/100-macos.conf line 1: Applying options for *
debug1: Reading configuration data /etc/ssh/crypto.conf
debug1: Authenticator provider $SSH_SK_PROVIDER did not resolve; disabling
debug1: Connecting to 192.168.45.82 [192.168.45.82] port 22327.
debug1: connect to address 192.168.45.82 port 22327: Connection refused
ssh: connect to host 192.168.45.82 port 22327: Connection refuse
```

원인은 두 가지였다. 과거에 `/etc/ssh/sshd_config` 에서 Port 22 를 Port 22327 로 매핑해두었고, `/etc/services` 에 22/tcp 를 22327/tcp 로 매핑해두었다. 위 두 가지를 해결하니 문제가 해결되었다.

## Fail2ban 세팅 🛡️

### fail2ban 설치 시 발생한 문제 & 해결법

fail2ban 서비스를 systemd 상에서 실행하려 하니 아래와 동일한 현상의 에러가 발생했다. 찾아보니 1.1.0 이하의 fail2ban 에 해당 문제가 있었고 최신 버전에 수정되었다고 한다. 어느 debian repo 도 최신 버전을 지원해주지 않고 있었다.[^3] [조언](https://discuss.frappe.io/t/fail2ban-fails-on-ubuntu-24-04-lts/127658)에 따라 아래와 같이 해결하였다.

```bash
# 직접 다운로드 하여 설치
cd /tmp
wget https://github.com/fail2ban/fail2ban/archive/refs/tags/1.1.0.tar.gz
tar -xzf 1.1.0.tar.gz
cd fail2ban-1.1.0
sudo python3 setup.py install

# 서비스 파일 활성화
cp build/fail2ban.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable fail2ban
systemctl start fail2ban
```

### fail2ban 설정

```bash
# jail.conf, jail.local 준비
/
❯ cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local
/                                                                                                                                                                                                root@pv/ /                                                                                                                                                                                                root@pve
❯ cp /etc/fail2ban/jail.local /etc/fail2ban/jail.local.backup
```

```bash
# jail.local 를 아래와 같이 설정

~                                                                                          root@pve
❯ cat /etc/fail2ban/jail.local
#
# /etc/fail2ban/jail.local
# Proxmox VE 강화 설정
# 작성일: 2025.12.30
#

[DEFAULT]
#
# 기본 설정
#

# 차단 시간 (기본)
bantime  = 1h

# 실패 카운트 기간
findtime = 10m

# 차단 전 허용 실패 횟수
maxretry = 5

# 로그 읽기 방식
backend  = systemd

# 차단 제외 IP (본인 네트워크 추가!)
ignoreip = 127.0.0.1/8 ::1 192.168.45.0/24

#
# 점진적 차단 시간 증가 (반복 공격자에게 더 긴 차단)
#
bantime.increment = true
bantime.factor    = 2
bantime.maxtime   = 4w

# 차단 방식 (nftables 사용 - Debian 12+ 권장)
banaction = nftables-multiport
banaction_allports = nftables-allports

#
# 이메일 알림 (선택사항 - 메일 서버 설정 필요)
#
# destemail = admin@example.com
# sender    = fail2ban@proxmox.local
# mta       = sendmail
# action    = %(action_mwl)s

# ============================================================
#                          JAILS
# ============================================================

#
# SSH 보호
#
[sshd]
enabled   = true
mode      = aggressive
port      = ssh
maxretry  = 3
findtime  = 10m
bantime   = 1h
# DDoS 공격 감지 포함 (aggressive 모드)

#
# Proxmox Web UI (pveproxy) 보호
#
[proxmox]
enabled   = true
port      = 8006
filter    = proxmox
maxretry  = 3
findtime  = 10m
bantime   = 1h

#
# Proxmox VNC/SPICE 프록시 보호
#
[proxmox-spice]
enabled   = true
port      = 3128,5900:5999
filter    = proxmox
maxretry  = 5
findtime  = 10m
bantime   = 30m

#
# 반복 범죄자 (Recidive)
# - fail2ban에 의해 여러 번 차단된 IP를 장기간 차단
#
[recidive]
enabled   = true
logpath   = /var/log/fail2ban.log
filter    = recidive
banaction = nftables-allports
maxretry  = 3
findtime  = 1d
bantime   = 1w

#
# SSH DDoS 공격 방어 (추가 보호)
#
[sshd-ddos]
enabled   = true
port      = ssh
filter    = sshd[mode=ddos]
maxretry  = 10
findtime  = 30s
bantime   = 10m

#
# 포트 스캔 감지 (portsentry 설치 시)
#
# [portsentry]
# enabled   = true
# logpath   = /var/lib/portsentry/portsentry.history
# maxretry  = 1
# bantime   = 1w

#
# Nginx (리버스 프록시 사용 시)
#
# [nginx-http-auth]
# enabled  = true
# port     = http,https
# logpath  = /var/log/nginx/error.log
# maxretry = 3
# bantime  = 1h

# [nginx-botsearch]
# enabled  = true
# port     = http,https
# logpath  = /var/log/nginx/access.log
# maxretry = 2
# bantime  = 1d

```
