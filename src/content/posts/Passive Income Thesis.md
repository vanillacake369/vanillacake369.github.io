| 분류 | 핵심 | 예시 |
| --- | --- | --- |
| Capital Leverage | 돈으로 돈의 노출을 키움 | 대출, 옵션, 선물, 마진, 레버리지 ETF |
| People Leverage | 전문가의 시간을 산다 | 컨설턴트, 대행사, 프리랜서, 협력사 |
| Worker Leverage | 직원으로 시간을 산다 | 사원 고용, 팀 운영 |
| Knowledge Leverage | 지식을 재사용한다 | 템플릿, 플레이북, 교육체계 |
| System Leverage | 프로세스를 자동화한다 | SOP, CRM, CI/CD, 자동화 봇 |
| Code Leverage | 소프트웨어가 일을 반복한다 | 스크립트, SaaS, API |
| Media Leverage | 콘텐츠가 사람을 모은다 | 블로그, 유튜브, 뉴스레터 |
| Network Leverage | 관계망이 기회를 확장한다 | 추천, 제휴, 커뮤니티 |
| Brand Leverage | 신뢰가 전환율을 높인다 | 유명 브랜드, 개인 브랜딩 |


# Theorem

개발자로 취직을 하게되어도 결국 자산 구조를 제대로 다잡지 않는다면
‘AI 가 나를 대체했을 때 나는 무엇인가? 나는 Income 을 어떻게 할 것인가’ 에 대한 대책이 없어진다
따라서 지금 내가 판단했을 때는 AI 공부보다는 Income 을 만드는 방법에 대해서 가장 먼저 공부할 필요가 있다고 본다
결국 기술 개발은 내가 흥미롭게 보는 것 중에 하나지만 이것도 어디까지나 밥벌이수단에 불과하니까,,,
