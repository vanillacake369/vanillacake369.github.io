# 📌 Why?

왜 배워야 하나?

---

RAID는 데이터 중복성(Redundancy), 성능 향상(Performance), 또는 두 가지 목적을 위해 여러 하드 드라이브를 하나의 논리 단위로 결합하는 기술입니다.

데이터 손실 방지, 시스템 가용성 증대, I/O 성능 최적화를 위해 기업 서버와 중요 시스템에서 필수적으로 사용됩니다.
**핵심 학습 이유:**

- 데이터 안정성 확보 (하드웨어 장애 대응)
- 시스템 다운타임 최소화
- 스토리지 성능 최적화
- 엔터프라이즈 인프라 이해

# 📖 What?

무엇을 배우나?

---

### 1. **Mirror (Mirroring) - RAID 1의 핵심 기술**

RAID 1 미러링은 두 개의 하드 드라이브에 실시간으로 데이터를 동일하게 복제하는 방식입니다.
**정의:**

- 모든 데이터를 2개 이상의 디스크에 완전히 복제
- 100% 데이터 중복성 제공

**특징:**

- 최소 디스크: 2개
- 용량 효율: 50% (2TB + 2TB = 2TB 사용 가능)
- 장애 허용: 1개 디스크 실패까지 견딤

### 2. **RAID 0 (Striping)**

RAID 0은 2개 이상의 디스크에 데이터를 균등하게 분산(스트라이핑)하며, 패리티 정보나 중복성, 결함 허용을 제공하지 않습니다.
**핵심 사양:**

- **최소 디스크:** 2개
- **용량:** 전체 디스크 합계 (2TB + 2TB = 4TB)
- **중복성:** 없음 (1개 디스크 실패 시 모든 데이터 손실)
- **성능:** 최고 (병렬 읽기/쓰기)

**트레이드오프:**

- ✅ 장점: 최고 속도, 100% 용량 활용
- ❌ 단점: 단일 디스크보다 위험, 데이터 보호 전무

**유스케이스:**

- 임시 캐시 서버
- 비중요 고속 스토리지
- 이미지/비디오 편집용 스크래치 디스크

### 3. **RAID 1 (Mirroring)**

RAID 1은 단일 디스크 장애 시 데이터 신뢰성을 제공하며, 한 드라이브가 실패해도 다른 드라이브에서 즉시 데이터를 이용할 수 있습니다.
**핵심 사양:**

- **최소 디스크:** 2개
- **용량:** 단일 디스크 크기 (2TB + 2TB = 2TB)
- **중복성:** 100% (완전 복제)
- **읽기 성능:** 향상 (두 디스크에서 동시 읽기 가능)
- **쓰기 성능:** 단일 디스크 수준

**트레이드오프:**

- ✅ 장점: 최고 신뢰성, 빠른 읽기, 단순한 복구
- ❌ 단점: 50% 용량 손실, 비용 2배

**유스케이스:**

- 중요 OS 부팅 드라이브
- 미션 크리티컬 데이터베이스
- 소규모 고가용성 시스템

### 4. **RAID 2 (Bit-level Striping with Hamming Code)**

RAID 2는 모든 데이터를 비트 레벨로 스트라이핑하며, 각 비트를 다른 드라이브에 기록하고, 오류 수정을 위해 해밍 코드를 사용합니다.
**핵심 사양:**

- 비트 단위 스트라이핑
- 해밍 코드 ECC (Error Correcting Code)
- 모든 디스크 동기화 필수

**현황:**

- ⚠️ **현재 사용 안 함** (Obsolete)
- 최신 HDD가 자체 ECC 내장으로 불필요

### 5. **RAID 3 (Byte-level Striping with Dedicated Parity)**

RAID 3는 바이트 레벨 스트라이핑과 전용 패리티 디스크를 사용하며, 순차적 데이터에서 높은 전송 속도를 보장합니다.
**핵심 사양:**

- **최소 디스크:** 3개
- 바이트 단위 스트라이핑
- 전용 패리티 디스크 1개
- 모든 디스크 동기 회전 필요

**트레이드오프:**

- ✅ 장점: 순차 읽기/쓰기 우수 (대용량 파일)
- ❌ 단점: 랜덤 I/O 성능 최악, 패리티 디스크 병목

**현황:**

- ⚠️ **거의 사용 안 함** (RAID 5로 대체됨)

**유스케이스 (과거):**

- 비압축 비디오 편집
- 대용량 순차 데이터 처리

### 6. **RAID 4 (Block-level Striping with Dedicated Parity)**

RAID 4는 블록 레벨 스트라이핑과 전용 패리티 디스크를 사용하며, 랜덤 읽기 성능은 좋지만 모든 패리티 데이터를 단일 디스크에 기록해야 해서 랜덤 쓰기 성능이 낮습니다.
**핵심 사양:**

- **최소 디스크:** 3개
- 블록 단위 스트라이핑 (16KB~128KB)
- 전용 패리티 디스크 1개

**트레이드오프:**

- ✅ 장점: 랜덤 읽기 좋음, 온라인 확장 가능
- ❌ 단점: 쓰기 시 패리티 디스크 병목

**현황:**

- ⚠️ **거의 사용 안 함** (RAID 5로 대체)
- NetApp이 독자 구현 사용 (RAID-DP)

### 7. **RAID 5 (Distributed Parity)**

RAID 5는 RAID 4와 비교하여 패리티를 분산 배치해 전용 패리티 디스크의 부하를 모든 RAID 멤버에 고르게 분산시킵니다.
**핵심 사양:**

- **최소 디스크:** 3개
- **용량:** (N-1) × 디스크 크기 (3×2TB = 4TB)
- **장애 허용:** 1개 디스크 실패
- 분산 패리티 (모든 디스크에 패리티 분산)

**트레이드오프:**

- ✅ 장점: 좋은 용량 효율, 비용 대비 신뢰성, 읽기 성능 우수
- ❌ 단점: 쓰기 오버헤드 (패리티 계산), 리빌드 시간 장기

**유스케이스:**

- 범용 파일 서버
- 중소 규모 데이터베이스
- NAS 시스템 (일반적)

### 8. **RAID 6 (Dual Distributed Parity)**

RAID 6는 블록 레벨 스트라이핑에 두 개의 패리티 블록을 모든 멤버 디스크에 분산 배치하여, 두 개의 디스크 장애를 허용합니다.
**핵심 사양:**

- **최소 디스크:** 4개
- **용량:** (N-2) × 디스크 크기 (6×2TB = 8TB)
- **장애 허용:** 2개 디스크 동시 실패 (N+2 중복성)
- 이중 패리티 (Reed-Solomon 알고리즘)

**트레이드오프:**

- ✅ 장점: 최고 신뢰성, 대용량 어레이에 적합
- ❌ 단점: 쓰기 성능 RAID 5보다 느림 (이중 패리티 계산), 2개 디스크 용량 손실

**유스케이스:**

- 대규모 스토리지 시스템 (EMC, IBM)
- 장기 아카이빙
- 높은 신뢰성 요구 환경
- 대용량 데이터베이스 (리빌드 실패 위험 감소)

### 9. **RAID 10 (1+0, Striped Mirrors)**

RAID 10은 RAID 0과 RAID 1의 조합으로, 먼저 디스크를 미러 쌍으로 구성한 후 이를 스트라이핑하여, RAID 0의 성능과 RAID 1의 장애 허용을 결합합니다.
**핵심 사양:**

- **최소 디스크:** 4개 (짝수 개수 필수)
- **용량:** 전체의 50% (4×2TB = 4TB)
- **장애 허용:** 미러 쌍당 1개 디스크 실패 (여러 디스크 동시 실패 가능, 같은 쌍만 아니면)
- 패리티 계산 없음

**트레이드오ფ:**

- ✅ 장점: 최고 성능 + 신뢰성, 빠른 리빌드, 패리티 오버헤드 없음
- ❌ 단점: 50% 용량 손실, 높은 비용

**유스케이스:**

- 고성능 데이터베이스 (OLTP)
- 가상화 환경
- 엔터프라이즈 애플리케이션 서버
- **"골드 스탠다드"** - 성능+신뢰성+용량 균형

# 💡 How?

어떻게 사용하나?

### **구현 방법**

1. **하드웨어 RAID**
2. **소프트웨어 RAID**
3. **펌웨어 RAID (Fake RAID)**

### **선택 가이드**

```
성능 최우선 + 예산 충분 → RAID 10
균형잡힌 선택 (범용) → RAID 5
대용량 + 높은 신뢰성 → RAID 6
OS/부팅 드라이브 → RAID 1
임시 캐시/스크래치 → RAID 0

```

### **주의사항**

⚠️ RAID는 백업 솔루션이 아닙니다.

데이터 보관 및 보안을 위한 백업 솔루션을 대체하지 않습니다.

- RAID는 하드웨어 장애만 보호
- 정기 백업 필수 (3-2-1 백업 규칙)
- 디스크 용량 증가로 리빌드 시간이 수 시간에서 수 일까지 소요될 수 있습니다.
- 대용량 디스크에서는 RAID 6 권장 (리빌드 중 추가 장애 위험)
