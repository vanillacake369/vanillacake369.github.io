## 핵심 팁

### 풀이

1.

문제가 무엇을 처리하고자 하는지의 본질을 찾는다
2.

바로 제출하지 말고 복잡도 분석을 통해 제한사항 및 비요구사항을 충족하는지 검사한다.
3. **한 문제 당 30분, 만약 통과하지 못 하면 바로 답지 보기**
4.

답지 본 이후에도 답지를 덮고 다시 풀어본다.

만약 못 풀면 이해하지 못 한 것이다.
5.

내 풀이방법 이외에도 다른 풀이법을 참고한다
6.

몇 일 몇 주 이후에 다시 푼다. 15 분 내로 풀 수 있을때까지 반복한다.
7.

회차 별로 어려움의 뎁스를 높인다

### 정리

1.

스스로 설명해보고 안 되면 이해가 안 된 것이므로 정리한다.
2.

왜 공부하는지 모를 때가 있으므로 이 알고리즘이 실제로 어디에 어떻게 쓰였는지 찾아본다.

# Study Notes 📓


## 이외에도,,,

```bash
// GCD / LCM
func gcd(a, b int) int { for b != 0 { a, b = b, a%b }; return a }
func lcm(a, b int) int { return a / gcd(a, b) * b }

// 순열/조합에서 결과 저장 시 반드시 copy!
tmp := make([]int, len(nums))
copy(tmp, nums)  // ★ append만 하면 참조 공유 버그
result = append(result, tmp)
```

## 데이터 범위에 따른 추천 알고리즘

> ⚠️ 이런 유형이 많았을뿐 반드시 해당 알고리즘들로 풀리지 않는다는 것을 명심

| **데이터 범위** | **추천 알고리즘** |
| --- | --- |
| N ≤ 10 | 완전 탐색 (재귀), 순열, 조합 |
| N ≤ 20 | DFS, 백트래킹 (타겟 넘버 같은 유형) |
| N ≤ 1,000 | O(N^2) 가능 (2중 for문, 간단한 DP) |
| N ≤ 100,000 | (N log N) 필수 (정렬, 이분 탐색, Heap) |
| N ≤ 1,000,000 | O(N) 또는 O(\log N) (그리디, 투 포인터) |

# Go 문법 및 유의사항

## 이외에도...

```java
// GCD / LCM
int gcd(int a, int b) { return b == 0 ? a : gcd(b, a % b); }
int lcm(int a, int b) { return a / gcd(a, b) * b; }

// 순열/조합에서 결과 저장 시 반드시 new ArrayList<>로 복사!
// ★ 참조만 넣으면 나중에 값이 바뀌는 버그
result.add(new ArrayList<>(current)); // ✅ 복사
// result.add(current);               // ❌ 참조 공유 버그
```


## 데이터 범위에 따른 추천 알고리즘

> ⚠️ 이런 유형이 많았을 뿐, 반드시 해당 알고리즘들로 풀리지 않는다는 것을 명심.
> 그냥 "어 이걸로 풀리지 않을까" 시도만 해볼 것!

| 데이터 범위 | 추천 알고리즘 |
| --- | --- |
| N ≤ 10 | 완전 탐색 (재귀), 순열, 조합 |
| N ≤ 20 | DFS, 백트래킹 (타겟 넘버 같은 유형) |

| N ≤ 100,000 | O(N log N) 필수 (정렬, 이분 탐색, Heap) |
| N ≤ 1,000,000 | O(N) 또는 O(log N) (그리디, 투 포인터) |

## 순열 (Permutation)

> 💡  "순서가 다르면 다른 것이다"

- **핵심 키워드:** `isUsed` (방문 체크), **순서 유지**.
- **설명 보완:** * 순열은 "뽑았던 숫자를 다시 쓸 수 없지만, **이전에 뽑았던 숫자의 순서가 바뀌면 새로운 경우**"로 간주합니다.

### **순열 풀이 예시 : **[**N과 M (1)**](https://www.acmicpc.net/problem/15649)** **

중복된 숫자 없는 순열에서 중요한 것은 다음 탐색 대상이 새로운 수여야 한다는 것이다.

즉 이전 탐색했던 숫자의 목록을 기억(memo) 해야한다.

이에 따라 1) 탐색 상태 저장공간 2) 탐색했던 숫자 목록을 기억해두는 저장공간 이
별도로 필요하다.

```go
package main

import (
	"bufio"
	"fmt"
	"os"
)

// 🤔 여기서 왜 arr, isUsed 를 10 길이의 배열로 삼음 ??
// 최대 범위 8 에서 +1 만큼의 배열을 선언해둬야
// IndexError 가 발생하지 않음
// 통상적으로 최대범위 +1, +2 만큼의 배열을 잡는 편임
var (
	n, m   int
	arr    [10]int
	isUsed [10]bool
	// 입력을 위한 리더와 출력을 위한 라이터 설정
	reader = bufio.NewReader(os.Stdin)
	writer = bufio.NewWriter(os.Stdout)
)

func main() {
	defer writer.Flush()
	fmt.Fscanf(reader, "%d %d
", &n, &m)

	permutation(0)
}

func permutation(index int) {
	// 📓 재귀 호출구조이니 반드시 base case 있어야 함
	// 모든 경우의 수를 출력하는 것이니
	// 재귀를 통해 순열 길이의 끝까지 탐색했다면
	// 출력하도록 함
	if index == m {
		for i := 0; i < m; i++ {
			fmt.Fprintf(writer, "%d ", arr[i])
		}
		fmt.Fprint(writer, "
")
		return
	}

	// 📓 순열 탐색 시작
	// Note :: 순열의 시작은 1 , 끝은 n 임
	for i := 1; i <= n; i++ {
		// 만약 탐색하지 않았다면 탐색 시작
		// 탐색을 완료한 숫자는 무시하도록 함
		if !isUsed[i] {
			// 순열의 순서인 index 에 해당하는 배열 값에
			// 탐색값을 삽입
			arr[index] = i
			// 탐색하였으므로 해당 숫자는 사용됨을 표시
			isUsed[i] = true
			// 현재 순열 순서의 다음 순번인
			// index + 1 에 대해서 탐색 시작
			permutation(index + 1)
			// i 로 시작한 순열 탐색이 끝나면
			// i + 1 로 시작한 순열 탐색에서 i 에 대한 사용표시를 위해
			// i 의 사용 표시를 제거
			// e.g. 1 로 시작한 순열 탐색 이후 2 로 시작한 순열탐색의 1 사용을 위해 1 사용표시 제해
			isUsed[i] = false
		}
	}
}

```

## 조합 (Combination)

"순서가 달라도 구성이 같으면 같은 것이다"

- **핵심 키워드:** `startIdx`, **오름차순 탐색**.
- **설명 보완:** * 조합은 "이미 뽑은 숫자의 **뒤에 있는 숫자들만** 뽑겠다"는 규칙을 정함으로써 중복을 원천 차단합니다.

## 제약 조건 탐색 (N-Queen 등)

- **핵심 키워드:** `Pruning` (가지치기), **유효성 검사**.
- **설명 보완:** * 이 유형은 단순히 모든 경우를 뽑는 게 아니라, 탐색 도중 **"이 길은 가봤자 답이 없다"**고 판단되면 즉시 되돌아오는 것이 핵심입니다.

### [N-Queen](https://www.acmicpc.net/problem/9663)

```go
package main

import (
	"bufio"
	"fmt"
	"math"
	"os"
)

var (
	// 입력을 위한 리더와 출력을 위한 라이터 설정
	reader = bufio.NewReader(os.Stdin)
	writer = bufio.NewWriter(os.Stdout)

	// 체스판 크기(input), 모든 경우의 수 결과개수(output)
	n        int
	possible int
)

func main() {
	defer writer.Flush()
	fmt.Fscanf(reader, "%d
", &n)

	// 탐색 state machine dynamic array 선언
	// 이미 이전 행에 대한 탐색 체크로 인해
	// 탐색 완료 여부를 별도로 체크할 필요 X
	board := make([]int, n)
	calcChessCase(0, board)

	fmt.Fprintf(writer, "%d", possible)
}

func calcChessCase(row int, board []int) {
	// Base case
	if row == n {
		possible++
		return
	}

	// Find all cases by recursive call tracking
	// Follow up untill i hits the bottom row
	for col := 0; col < n; col++ {
		if isNotQueenPath(row, col, board) {
			// 퀸 배치
			board[row] = col
			// 다음 행에 대한 탐색 시작
			calcChessCase(row+1, board)
		}

	}
}

// 퀀 경로에 있는지 확인
// 1) 같은 열 여부
// 2) 대각선 여부
func isNotQueenPath(row int, col int, board []int) bool {
	for i := 0; i < row; i++ {
		if board[i] == col || math.Abs(float64(i-row)) == math.Abs(float64(board[i]-col)) {
			return false
		}
	}
	return true
}

```
