## 해시

- Map 순회 순서는 랜덤 — 순서 필요하면 키 슬라이스 별도 정렬
- **키로 슬라이스 불가** → `[2]int` 배열 또는 `fmt.Sprintf` 문자열로 변환

```bash
// 빈도수 세기
freq := make(map[int]int)
for _, v := range arr { freq[v]++ }

// Two Sum: 보수(complement)를 해시에서 찾기
seen := make(map[int]int)
for i, num := range nums {
    if j, ok := seen[target-num]; ok { return [2]int{j, i} }
    seen[num] = i
}

// 좌표 키
visited := make(map[[2]int]bool)
visited[[2]int{x, y}] = true
```

## 구현

- **최적화보다 정확성 우선** — 문제를 그대로 코드로
- **경계 조건** — 0-indexed vs 1-indexed, 배열 범위
- **모듈러 연산** — 중간 계산마다 `% MOD`

투포인터

```bash
// 정렬된 배열, 양 끝에서 좁혀오기
lo, hi := 0, len(arr)-1
for lo < hi {
    sum := arr[lo] + arr[hi]
    if sum == target { return }
    if sum < target { lo++ } else { hi-- }
}
```

슬라이딩 윈도우

```bash
// 길이 k 윈도우의 최대 합
windowSum := 0
for i := 0; i < k; i++ { windowSum += arr[i] }
maxSum := windowSum
for i := k; i < len(arr); i++ {
    windowSum += arr[i] - arr[i-k]  // 오른쪽 추가, 왼쪽 제거
    if windowSum > maxSum { maxSum = windowSum }
}
```

**그리디**

```bash
// 회의실 배정: 끝나는 시간 기준 정렬 → 빨리 끝나는 것부터
sort.Slice(meetings, func(i, j int) bool {
    return meetings[i][1] < meetings[j][1]
})
count, lastEnd := 0, 0
for _, m := range meetings {
    if m[0] >= lastEnd { count++; lastEnd = m[1] }
}
```

## 정렬

```bash
import "sort"

// 내림차순 (큰 놈이 앞)
sort.Slice(priorities, func(i, j int) bool {
    return priorities[i] > priorities[j]
})

// 구조체 정렬 (우선순위 내림차순, 같으면 인덱스 오름차순)
sort.Slice(items, func(i, j int) bool {
    if items[i].priority == items[j].priority {
        return items[i].index < items[j].index
    }
    return items[i].priority > items[j].priority
})
```

```bash
stack := []int{}

// Push
stack = append(stack, 1)

// Pop
if len(stack) > 0 {
    top := stack[len(stack)-1]    // Top 확인
```

```bash
queue := []Item{}

queue = append(queue, Item{3, 0})

if len(queue) > 0 {
```

## 해시 (HashMap)

- Map 순회 순서는 보장되지 않음 → 순서 필요하면 키 리스트 별도 정렬
- 키로 배열 불가 → `Arrays.toString()`, `"x,y"` 문자열, 또는 `Map<Long, V>` (좌표 인코딩)

```java
// 빈도수 세기
Map<Integer, Integer> freq = new HashMap<>();
for (int v : arr) {
    freq.merge(v, 1, Integer::sum); // 또는 freq.put(v, freq.getOrDefault(v, 0) + 1)
}

// Two Sum: 보수(complement)를 해시에서 찾기
Map<Integer, Integer> seen = new HashMap<>();
for (int i = 0; i < nums.length; i++) {
    int complement = target - nums[i];
    if (seen.containsKey(complement)) {
        return new int[]{seen.get(complement), i};
    }
    seen.put(nums[i], i);
}

// 좌표 키 — 문자열 인코딩
Set<String> visited = new HashSet<>();
visited.add(x + "," + y);

// 좌표 키 — long 인코딩 (더 빠름)
Set<Long> visited = new HashSet<>();
visited.add((long) x * 100001 + y); // y 범위에 맞게 조절
```

- 최적화보다 정확성 우선 — 문제를 그대로 코드로
- 경계 조건 — 0-indexed vs 1-indexed, 배열 범위
- 모듈러 연산 — 중간 계산마다 `% MOD`

### 투 포인터

```java
// 정렬된 배열, 양 끝에서 좁혀오기
int lo = 0, hi = arr.length - 1;
while (lo < hi) {
    int sum = arr[lo] + arr[hi];
    if (sum == target) return;
    if (sum < target) lo++;
    else hi--;
}
```

### 슬라이딩 윈도우

```java
// 길이 k 윈도우의 최대 합
int windowSum = 0;
for (int i = 0; i < k; i++) windowSum += arr[i];
int maxSum = windowSum;
for (int i = k; i < arr.length; i++) {
    windowSum += arr[i] - arr[i - k]; // 오른쪽 추가, 왼쪽 제거
    maxSum = Math.max(maxSum, windowSum);
}
```

### 그리디

```java
// 회의실 배정: 끝나는 시간 기준 정렬 → 빨리 끝나는 것부터
Arrays.sort(meetings, (a, b) -> a[1] - b[1]);
int count = 0, lastEnd = 0;
for (int[] m : meetings) {
    if (m[0] >= lastEnd) {
        count++;
        lastEnd = m[1];
    }
}
```

## 1. 구현 / 시물레이션 (최다 기출 유형)

> 📌 keyword

구현/ 시뮬레이션 문제는 실제로 문제에서 주어진 상황을 **정확히 읽고**, **조건대로 코드를 작성하는 능력**을 평가합니다.
이 유형은 코딩테스트에서 가장 빈출되며, ‘어려운 알고리즘은 아닌데 시간이 오래 걸린다’, ‘실수하면 틀린다’는 피드백이 자주 나옵니다.

| **유형** | **설명** |
| --- | --- |
| **시간 계산 시뮬레이션** | 시계 돌리기, 타이머, 주기 변화 등 (예: 24시간 → 분 변환) |
| **보드 이동** | 격자 내 캐릭터 이동, 충돌, 회전 등 |
| **문자열 파싱 및 조작** | 쉼표, 공백 등 기준으로 분할, 특수 조건 추출 |
| **명령어 처리 시뮬레이션** | 스택, 큐, 힙처럼 명령에 따라 자료구조 조작 |
| **이벤트 스케줄링 시뮬레이션** | 우선순위/시간 조건대로 동작 |

2. 문제 많이 풀기

## Stack

### 테크닉 : **Monotonic Stack, 순서 유지하면서 다음 큰 원소 찾기**

[https://justicehui.github.io/medium-algorithm/2019/01/01/monotoneStack/](https://justicehui.github.io/medium-algorithm/2019/01/01/monotoneStack/)

> 💡 핵심 원리
> Tip 💡

```java
// Next Greater Element (오큰수) 템플릿: 오른쪽에서 처음 만나는 더 큰 값
static int[] nextGreater(int[] a) {
    int n = a.length;
    int[] ans = new int[n];
    Arrays.fill(ans, -1);
    Deque<Integer> st = new ArrayDeque<>();

    for (int i = 0; i < n; i++) {
        while (!st.isEmpty() && a[st.peek()] < a[i]) {
            ans[st.pop()] = a[i];
        }
        st.push(i);
    }
    return ans;
}
```

### 문제 : [탑](https://www.acmicpc.net/problem/2493)

```java
private static int[] resolve(int n, int[] inputs) {
    int[] answer = new int[n];
    Deque<int[]> stack = new ArrayDeque<>();

    for (int i = 0; i < n; i++) {
        int currentHeight = inputs[i];
        int currentIndex = i + 1;

        while (!stack.isEmpty() && stack.peek()[1] < currentHeight) {
            stack.pop();
        }

        if (!stack.isEmpty()) {
            answer[i] = stack.peek()[0];
        } else {
            answer[i] = 0;
        }

        stack.push(new int[] {currentIndex, currentHeight});
    }

    return answer;
}
```

### 문제 : [오큰수](https://www.acmicpc.net/problem/17298)

```java
private static int[] resolve(int n, int[] inputs) {
    int[] result = new int[n];

    Deque<Integer> stack = new ArrayDeque<>();
    for (int i = 0; i < result.length; i++) {
        int currentValue = inputs[i];

        while (!stack.isEmpty() && inputs[stack.peek()] < currentValue) {
            int previousIndex = stack.pop();
            result[previousIndex] = currentValue;
        }
        stack.push(i);
    }

    while (!stack.isEmpty()) {
        result[stack.pop()] = -1;
    }
    return result;
}
```

```java
private static long resolve(int n, int[] inputs) {
    long result = 0;
    Deque<Integer> stack = new ArrayDeque<>();

    for (int i = 0; i < inputs.length; i++) {
        int currentHeight = inputs[i];
        while (!stack.isEmpty() && stack.peek() <= currentHeight) {

        }
        result += stack.size();
        stack.push(inputs[i]);
    }
    return result;
```

### 문제 : [오아시스 재결합](https://www.acmicpc.net/problem/3015)

```java

/**
 * # 해석
 * 두 위치의 수 사이에 대해 조건에 맞는지 판별
 *
 * # 접근
 * 루프를 통해 접근 시 N 이 500,000 이므로 시간복잡도 초과
 * 따라서 LIFO 기반 수를 순서대로 입력하고
 * 내가 누군가에게 보여질 수 있는지 조건을 판별
 *
 * # 상태값
 * 이전의 수들의 볼 수 있는 상태를 연속해서 가져가야하므로
 * {수의 크기, 볼 수 있는 횟수} 를 가져가서
 * 이전 수들의 볼 수 있는 횟수를 캐싱
 */
private static long resolve(int n, int[] inputs) {
    long sum = 0;

    Deque<int[]> stack = new ArrayDeque<>();

    for (int i = 0; i < n; i++) {
        int currentCache = 1;
        while (!stack.isEmpty()) {
            int[] prevPerson = stack.peek();
            if (prevPerson[0] < inputs[i]) {
                sum += prevPerson[1];
                stack.pop();
                continue;
            }
            if (prevPerson[0] == inputs[i]) {
                sum += prevPerson[1];
                currentCache = prevPerson[1] + 1;
                stack.pop();
                continue;
            }
            sum++;
            break;
        }
        stack.push(new int[] {inputs[i], currentCache});
    }

    return sum;
}

```
