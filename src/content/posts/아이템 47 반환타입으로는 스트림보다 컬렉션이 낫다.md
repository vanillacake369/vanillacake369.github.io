# Stream does not support Iterable

---

Collection은 Iterable을 지원하나, Stream은 그렇지 않다. 
이는 Collection과 Stream의 구조를 뜯어보면 알 수 있다.

## Adapter method : Stream → Iterable

---

<details>
<summary>어댑터??</summary>

> 특정 인터페이스를 지원하지 않는 대상 객체를 인터페이스를 지원하는 Adapter에 집어넣어서 사용하는 방법

[https://johngrib.github.io/wiki/pattern/adapter/](https://johngrib.github.io/wiki/pattern/adapter/)

![](/images/notion/005922d155a15365.svg)

다음은 헤드 퍼스트 디자인 패턴의 예제이다

- 다음은 오리를 표현한 것이다.

```java
interface Duck {
  public void quack();  // 오리는 꽉꽉 소리를 낸다
  public void fly();
}

class MallardDuck implements Duck {
  @Override
  public void quack() {
    System.out.println("Quack");
  }
  @Override
  public void fly() {
    System.out.println("I'm flying");
  }
}
```

- 다음은 칠면조를 표현한 것이다. 두 새의 인터페이스가 다르다는 점에 주목.

```java
interface Turkey {
  public void gobble();   // 칠면조는 골골거리는 소리를 낸다
  public void fly();
}

class WildTurkey implements Turkey {
  @Override
  public void gobble() {
    System.out.println("Gobble gobble");
  }
  @Override
  public void fly() {
    System.out.println("I'm flying a short distance");
  }
}
```

- 그리고 이것이 바로 어댑터이다.

```java
class TurkeyAdapter implements Duck {
  Turkey turkey;

  public TurkeyAdapter(Turkey turkey) {
    this.turkey = turkey;
  }
  @Override
  public void quack() {
    turkey.gobble();
  }
  @Override
  public void fly() {
    // 칠면조는 멀리 날지 못하므로 다섯 번 날아서 오리처럼 긴 거리를 날게 한다
    for (int i = 0; i < 5; i++) {
      turkey.fly();
    }
  }
}
```

이 어댑터를 다이어그램으로 표현하자면 다음과 같다. 어댑터 `TurkeyAdapter`를 통해 `Turkey`를 `Duck`처럼 사용할 수 있게 된 것이다.

![](/images/notion/a5c5ddde069acf1e.svg)
</details>

자바에서 implements를 지원해주고 있지 않지만, 어댑터를 직접 작성해줌으로서 변환해줄 수 있다.

```java
// Adapter from Stream<E> to Iterable<E>
public static <E> Iterable<E> iterableOf(Stream<E> stream) {
    return stream::iterator;
}
```

아래 그림처럼, Stream의 최상위 BaseStream에서는 Iterator에 대한 메서드를 지원해주고 있다.

![](/images/notion/12affdce8a4f8c62.png)



## Adapter method : Iterable → Stream

---

반대로, Iterable를 Stream으로 변환이 가능하다.
물론 이를 우리가 직접 작성해주어야한다.

```java
// Adapter from Iterable<E> to Stream<E>
public static <E> Stream<E> streamOf(Iterable<E> iterable) {
    return StreamSupport.stream(iterable.spliterator(), false);
}
```

# 아니, 그러면 어떤 반환타입을 써야해? 🤔

저자의 의견

나의 의견

> Public API라면 Collection을 반환하게 해주자.

## 왜??

### Collection 인터페이스는 Iterable과 stream을 모두 제공한다.

Collection은 Iterable을 지원하나, Stream은 그렇지 않다.
반면, Collection은 Iterable의 하위 타입이며, stream()메서드를 통해 본인에 대한 Stream 객체를 반환해줄 수 있다.
따라서 어떤 목적으로 어떤 개발자가 사용할지 모르는 Public API라면 이 방법을 채택하자.


### Stream만을 사용하는 경우, 굳이 Iterable에 대한 어댑터를 만들 필요가 없다.

매 번 아래의 메소드를 호출하여 변환을 할 이유가 없다면, 
스트림을 반환하여 Stream API의 효율성을 극대화 해주는 것도 하나의 방법이다.

```java
// Adapter from Iterable<E> to Stream<E>
public static <E> Stream<E> streamOf(Iterable<E> iterable) {
    return StreamSupport.stream(iterable.spliterator(), false);
}
```
