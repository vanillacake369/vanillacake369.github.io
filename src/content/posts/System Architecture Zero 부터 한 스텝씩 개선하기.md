---

# What? 뭘 배움?

---

### 단일 서버

로직처리, 데이터저장 등등 모든 일을 하나의 서버에서 맡아서 하는 아키텍처이다.

### 단일 서버의 한계점

사용자가 많아져서 요청이 증진되면, 용도에 따라 서버를 증진한다.
각각의 용도에 따라 서버를 분리하기도 한다.

- 트래픽 처리 용도
- 스케줄링 처리 용도
- DB 서버

### 어떤 종류의 DB를 선택할 것인가?

보통 관계형 DB를 사용하여 데이터 간의 연관관계를 표현하곤 한다.

### 그렇담 어떻게 서버를 증진할 것인가?

- 수직적 확장 == 스케일 업
- 수평적 확장 == 스케일 아웃

> 수직적 확장의 한계점

### 장애 발생을 위한 기술적용사항

- 로드 밸런서
- DB 다중화

### 성능 개선을 위한 기술적용사항

- 캐시 계층
- CDN 사용
- 메세지 큐

### DB 수직적 확장

- DB 서버 자체를 업그레이드 << 고사양 CPU, RAM 등등을 추가

> SPOF 에 대한 대책방안이 힘듬

### DB 수평적 확장

- 샤딩 :: DB를 샤드라는 작은 단위로 분할한다.
- 다만 샤딩을 도입하면 풀어야할 숙제가 늘어난다.
